
# 📘 Detailed Notes on `BufferedReader` in Java

In Java, `BufferedReader` is a class used to read text/input efficiently.

It is mainly used for:

* 🚀 Fast input
* 📂 File reading
* 🧠 Competitive Programming
* ⚡ Performance optimization

---

# 📌 What is `BufferedReader`?

`BufferedReader` is a class from:

```java id="u7m2q5"
java.io package
```

used to read:

* text
* characters
* lines

efficiently using:

# Buffer Memory

---

# 📌 Import Statements

```java id="k4n8v1"
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.FileReader;
```

---

# 📌 Why is it called "Buffered" Reader? 🤔

Because it uses:

# Buffer Memory

instead of reading data one character at a time.

---

# 📌 What is Buffer Memory?

A:

# buffer

is temporary memory used to store data before processing.

---

# 🔥 Without Buffer

```text id="m1v7q3"
Keyboard → JVM → Program
```

Many small slow operations happen.

---

# 🔥 With Buffer

```text id="x8m2k6"
Keyboard → Buffer Memory → Program
```

Data is read in larger chunks, making input much faster.

---

# 📌 Why Use `BufferedReader`?

Main advantages:

✅ Faster than Scanner
✅ Efficient for large input
✅ Used in Competitive Programming
✅ Better performance
✅ Good for file reading

# 📌 Example 1 — Reading String from Keyboard

```java id="y4m8q1"
import java.io.BufferedReader;
import java.io.InputStreamReader;

public class Main {

    public static void main(String[] args) throws Exception {

        BufferedReader br =
                new BufferedReader(
                        new InputStreamReader(System.in));

        System.out.print("Enter Name: ");

        String name = br.readLine();

        System.out.println("Name is: " + name);
    }
}
```

# 📌 Why `BufferedReader` is Faster Than Scanner? 🚀

## Scanner

Scanner does:

* regex processing
* datatype parsing
* tokenization

which makes it slower.

---

# 🔥 BufferedReader

BufferedReader:

* simply reads raw text
* uses buffer memory
* minimal processing

So performance becomes much faster.

---

# 📌 Example Program

```java id="k9v3m1"
import java.io.*;

public class Main {

    public static void main(String[] args) throws Exception {

        BufferedReader br =
                new BufferedReader(
                        new FileReader("input.txt"));

        String line;

        while((line = br.readLine()) != null){

            System.out.println(line);
        }

        br.close();
    }
}
```

# 📌 Competitive Programming Standard

Most CP coders use:

```java id="v5m1k8"
BufferedReader + StringTokenizer
```

because:

* fastest input
* efficient parsing

---

# Example

```java id="x3n7q1"
import java.io.*;
import java.util.StringTokenizer;

public class Main {

    public static void main(String[] args) throws Exception {

        BufferedReader br =
                new BufferedReader(
                        new InputStreamReader(System.in));

        StringTokenizer st =
                new StringTokenizer(br.readLine());

        int a = Integer.parseInt(st.nextToken());
        int b = Integer.parseInt(st.nextToken());

        System.out.println(a + b);
    }
}
```

---

# 📌 Why StringTokenizer Faster Than split()?

Because:

```java id="w8m2v4"
split()
```

uses regex internally (slower).

`StringTokenizer` is lightweight and faster.
