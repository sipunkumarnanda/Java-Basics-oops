
# 🚨 Detailed Notes on `throws Exception` in Java

In Java, exceptions are used to handle runtime problems/errors that may occur during program execution.

One of the most commonly used exception keywords is:

```java id="m8q2v5"
throws Exception
```

This topic is extremely important for:

* Java Basics
* File Handling
* JDBC
* Networking
* Interviews
* Backend Development

---

# 📌 What is an Exception?

An exception is:

> ❌ An unexpected problem/error that occurs during program execution and interrupts the normal flow of the program.

---

# 🔥 Real-Life Example

Imagine:

```text id="v7n4m1"
You are watching YouTube
        ↓
Internet disconnects
        ↓
Video stops unexpectedly
```

That interruption is similar to an:

# Exception

---

# 📌 Examples of Exceptions in Java

| Situation             | Exception                      |
| --------------------- | ------------------------------ |
| Divide by zero        | ArithmeticException            |
| File not found        | FileNotFoundException          |
| Invalid array index   | ArrayIndexOutOfBoundsException |
| Accessing null object | NullPointerException           |
| Input/output issue    | IOException                    |

---

# ⚡ Example 1 — Exception

```java id="t5m9q2"
int a = 10 / 0;
```

Output:

```text id="x2r7v4"
ArithmeticException
```

because division by zero is illegal.

---

# 📌 Why Exceptions Exist?

Exceptions help Java:

* stop unsafe execution
* prevent crashes
* notify developers about problems
* make programs robust

---

# 📌 What is `throws Exception`?

```java id="u8n3m5"
throws Exception
```

means:

> “This method may generate an exception/error, and I am passing the responsibility of handling it to the caller.”

---

# 📌 Basic Syntax

```java id="g4v1k9"
returnType methodName(parameters) throws Exception {

}
```

Example:

```java id="m1r8q3"
public static void main(String[] args) throws Exception {

}
```

---

# 📌 What is a Caller? 🤔

This is VERY important.

---

# 🔥 Caller Meaning

A:

# caller

is:

> the method/function that calls another method.

---

# Example

```java id="d7m2v8"
class Test {

    static void show() {

        System.out.println("Hello");
    }

    public static void main(String[] args) {

        show();
    }
}
```

---

# Here:

```java id="w5n9k1"
show();
```

means:

* `main()` called `show()`

So:

| Method   | Role          |
| -------- | ------------- |
| `main()` | Caller        |
| `show()` | Called method |

---

# 📌 Internal Flow

```text id="p9v4m2"
main()
   ↓ calls
show()
```

So:

* `main()` = caller
* `show()` = callee/called method

---

# 📌 Why `throws Exception` Needed?

Some operations are risky ⚠️

Examples:

* reading files
* database connection
* network communication
* user input

Java forces you to handle possible failures.

---

# Example — File Reading

```java id="r2m8q6"
BufferedReader br =
        new BufferedReader(
                new FileReader("input.txt"));
```

Possible problems:

* file may not exist
* permission denied
* disk issue

So Java says:

> “Handle possible exceptions.”

---

# 📌 Two Ways to Handle Exceptions

## ✅ 1. try-catch

OR

## ✅ 2. throws

---

# ✅ Method 1 — try-catch

```java id="x7n1v4"
try {

    BufferedReader br =
            new BufferedReader(
                    new FileReader("input.txt"));

} catch (Exception e) {

    System.out.println("Error occurred");
}
```

---

# 🔥 What Happens Here?

If error occurs:

* control jumps to `catch`
* program handles error safely

---

# 📌 Meaning

```text id="y4m9q1"
"I will handle the error myself."
```

---

# ✅ Method 2 — throws Exception

```java id="v8k2m5"
public static void main(String[] args) throws Exception {

    BufferedReader br =
            new BufferedReader(
                    new FileReader("input.txt"));
}
```

---

# 🔥 What Happens Here?

You are saying:

```text id="k1r7v3"
"I am NOT handling the error here.
Let the caller handle it."
```

---

# 📌 Internal Flow

```text id="m5x2n8"
Method
   ↓
Exception occurs
   ↓
throws Exception
   ↓
Caller handles it
```

---

# 📌 But Who Is Caller of `main()`? 🤔

Excellent question.

For:

```java id="r7m1v9"
public static void main(String[] args)
```

the caller is:

# JVM (Java Virtual Machine)

---

# Internal Flow

```text id="p4k8n2"
JVM
 ↓ calls
main()
```

So if:

```java id="t9m3v5"
main() throws Exception
```

then:

* exception goes to JVM
* JVM handles/displays it

---

# Example

```java id="u3n7m1"
public class Test {

    public static void main(String[] args) throws Exception {

        throw new Exception("Something went wrong");
    }
}
```

Output:

```text id="y6k2v8"
Exception in thread "main" java.lang.Exception:
Something went wrong
```

because JVM handled it.

# 📌 Why Compiler Forces Exception Handling?

Example:

```java id="k8m3v1"
BufferedReader br =
        new BufferedReader(
                new FileReader("abc.txt"));
```

Compiler error:

```text id="w2n7q5"
Unhandled exception type FileNotFoundException
```

because Java wants safer code.

---

# 📌 Checked vs Unchecked Exceptions

# ✅ 1. Checked Exceptions

Compiler FORCES handling.

Examples:

* IOException
* SQLException
* FileNotFoundException

Need:

```java id="m7v2k9"
try-catch
```

OR

```java id="p1n8m4"
throws
```

---

# ✅ 2. Unchecked Exceptions

Compiler does NOT force handling.

Examples:

* ArithmeticException
* NullPointerException
* ArrayIndexOutOfBoundsException

---

# 📌 Why `BufferedReader` Needs `throws`?

Methods like:

```java id="t4m1q7"
readLine()
```

may throw:

```java id="v9n2k5"
IOException
```

So Java forces handling.

---

# 📌 How to Use `throws Exception`

## Syntax

```java id="d6m8v3"
methodName() throws Exception
```

---

# Example

```java id="r3n7k1"
import java.io.*;

public class Test {

    public static void main(String[] args) throws Exception {

        BufferedReader br =
                new BufferedReader(
                        new InputStreamReader(System.in));

        String name = br.readLine();

        System.out.println(name);
    }
}
```

---

# 📌 Custom Method Example

```java id="k5v1m8"
class Test {

    static void show() throws Exception {

        System.out.println("Hello");
    }

    public static void main(String[] args) throws Exception {

        show();
    }
}
```

# 📌 Difference Between `throw` and `throws`

| Keyword  | Purpose                      |
| -------- | ---------------------------- |
| `throw`  | actually throws exception    |
| `throws` | declares possible exceptions |
