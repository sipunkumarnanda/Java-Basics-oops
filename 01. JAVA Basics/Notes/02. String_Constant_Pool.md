
# String Immutability in Java — Detailed Notes

## What is a String in Java?

In Java, a `String` is an object used to store text.

Example:

```java
String name = "Sipun";
```

---

# What Does Immutable Mean?

## Definition

A String is **immutable** in Java.

Immutable means:

> Once a String object is created, its content cannot be changed.

You cannot directly modify:

* characters
* length
* content

of an existing String object.

---

# Example 1 — Basic Immutability

```java
String s1 = "Sipun";

s1 = "Sipun Kumar";

System.out.println(s1);
```

## Output

```text
Sipun Kumar
```

---

# Important Doubt

You may think:

> “Did Java modify the original String?”

Answer:

❌ NO

Java did NOT change `"Sipun"`.

Instead:

* Java created a NEW String object `"Sipun Kumar"`
* `s1` now points to the new object

---

# Internal Memory Representation

## Step 1

```java
String s1 = "Sipun";
```

Memory:

```text
s1 ----> "Sipun"
```

---

## Step 2

```java
s1 = "Sipun Kumar";
```

Java creates a NEW object.

Now memory becomes:

```text
"Sipun"          (old object)

s1 ----> "Sipun Kumar"   (new object)
```

The old String remains unchanged.

This proves immutability.

---

# Why Are Strings Immutable?

Java designers made Strings immutable because of:

* Security
* Thread safety
* Performance
* String Pool optimization

---

# String Pool Concept

Java stores string literals inside a special memory area called:

# String Constant Pool (SCP)

---

# Example 2 — Same Object Reuse

```java
String a = "apple";
String b = "apple";

System.out.println(a == b);
System.out.println(a.equals(b));
```

## Output

```text
true
true
```

---

# Your Doubt

## Why is `a == b` true?

### Explanation

Java checks:

> “Does `"apple"` already exist in the String Pool?”

If YES:

* reuse existing object
* do not create another object

So both variables point to the SAME object.

---

# Internal Memory

```text
a ----\
       ---> "apple"
b ----/
```

Only ONE object exists.

That is why:

```java
a == b
```

returns:

```text
true
```

because both references point to the same memory location.

---

# What Does `==` Mean?

`==` compares:

# memory/reference

NOT content.

---

# What Does `.equals()` Mean?

`.equals()` compares:

# actual content/value

---

# Example

```java
String a = "apple";
String b = "apple";

System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true
```

Why?

| Expression    | Meaning       | Result |
| ------------- | ------------- | ------ |
| `a == b`      | same object?  | true   |
| `a.equals(b)` | same content? | true   |

---

# Example 3 — Using `new String()`

```java
String a = new String("Hello");
String b = new String("Hello");

System.out.println(a == b);
System.out.println(a.equals(b));
```

## Output

```text
false
true
```

---

# Why is `a == b` false here?

Because `new String()` forces Java to create NEW objects every time.

Memory:

```text
a ----> "Hello"   (Object 1)

b ----> "Hello"   (Object 2)
```

Different objects ⇒ different memory locations.

So:

```java
a == b
```

becomes:

```text
false
```

---

# Why is `.equals()` still true?

Because contents are same:

```text
"Hello" == "Hello"
```

So:

```java
a.equals(b)
```

returns:

```text
true
```

---

# Your Main Doubt — If Strings Share Same Object, What Happens When Modified?

Example:

```java
String a = "apple";
String b = "apple";

a = "mango";
```

---

# Important Question

Did Java modify `"apple"` into `"mango"`?

❌ NO

Because Strings are immutable.

Java CANNOT change existing String objects.

---

# What Actually Happens?

Java creates a NEW object:

```text
"mango"
```

and `a` starts pointing there.

---

# Memory Before Modification

```text
a ----\
       ---> "apple"
b ----/
```

---

# Memory After Reassignment

```text
b ------> "apple"

a ------> "mango"
```

Notice:

* `"apple"` remains unchanged
* only reference `a` changed

This is the REAL meaning of immutability.

---

# Example 4 — Full Proof

```java
String a = "apple";
String b = "apple";

System.out.println(a == b); // true

a = "mango";

System.out.println(a); // mango
System.out.println(b); // apple

System.out.println(a == b); // false
```

---

# Why Final Output Becomes False?

After reassignment:

```text
a ----> "mango"

b ----> "apple"
```

Different objects now.

So:

```java
a == b
```

becomes:

```text
false
```

---

# Can We Modify Characters Directly?

Example:

```java
String s = "Sipun";

s.charAt(1) = 'a';
```

❌ INVALID

Compilation error.

---

# Why?

Because:

* `charAt()` only RETURNS a character
* Strings cannot be modified directly

---

# Correct Way

Create a NEW String.

```java
String s = "Sipun";

s = "S" + 'a' + s.substring(2);

System.out.println(s);
```

## Output

```text
Sapun
```

Again:

* old String unchanged
* new String created

---

# How to Verify New Object Creation?

Using:

```java
System.identityHashCode()
```

Example:

```java
String s1 = "Sipun";

System.out.println(System.identityHashCode(s1));

s1 = "Sipun Kumar";

System.out.println(System.identityHashCode(s1));
```

Different hash codes indicate different objects.

---

# String Pool vs Heap

| Creation Type         | Memory      |
| --------------------- | ----------- |
| `"hello"`             | String Pool |
| `new String("hello")` | Heap        |

---

# Best Practice

## Always Use `.equals()` for String Comparison

Correct:

```java
a.equals(b)
```

Avoid:

```java
a == b
```

unless checking references intentionally.

# Final Mental Model

Think of String objects like permanent printed paper.

You cannot erase/change text on the paper.

If you want new text:

* create a NEW paper
* point to the new one

That is exactly how Java Strings work.
