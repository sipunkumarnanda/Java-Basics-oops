
# Detailed Notes on `String[] args` in Java

In Java, every Java program starts execution from:

```java id="y7thc5"
public static void main(String[] args)
```

This is called the:

# Main Method

Java Virtual Machine (JVM) looks for this method to start the program.

---

# Syntax Breakdown

```java id="4plpxv"
public static void main(String[] args)
```

| Keyword         | Meaning                                 |
| --------------- | --------------------------------------- |
| `public`        | JVM can access it from anywhere         |
| `static`        | JVM can call it without creating object |
| `void`          | returns nothing                         |
| `main`          | entry point of program                  |
| `String[] args` | stores command-line arguments           |

---

# What is `String[] args`?

`args` is:

# an array of Strings

used to store:

# command-line arguments

passed while running the Java program.

---

# What Are Command-Line Arguments?

These are values given from terminal/command prompt BEFORE the program starts.

Example:

```bash id="g13x7g"
java Main Hello 25 Java
```

Here:

* `"Hello"`
* `"25"`
* `"Java"`

are command-line arguments.

Java stores them inside:

```java id="qf2v1i"
String[] args
```

---

# Internal Working

Command:

```bash id="zrfuw5"
java Main Hello 25 Java
```

Internally Java creates:

```java id="3wyb0t"
args[0] = "Hello"
args[1] = "25"
args[2] = "Java"
```

Memory representation:

```text id="5nd57g"
args ----> ["Hello", "25", "Java"]
```

---

# Example 1 — Basic Usage

```java id="0o4h5k"
public class Main {
    public static void main(String[] args) {

        System.out.println(args[0]);
        System.out.println(args[1]);
    }
}
```

Run:

```bash id="7hfe4w"
java Main Sipun Kumar
```

Output:

```text id="ecg5me"
Sipun
Kumar
```

---

# Important Point

Everything inside `args` is stored as:

# String

Even numbers.

Example:

```bash id="y0g0bp"
java Main 10 20
```

Internally:

```java id="3rv6pc"
args[0] = "10"
args[1] = "20"
```

NOT integers.

---

# Converting String to Integer

Use:

```java id="0g6nhv"
Integer.parseInt()
```

Example:

```java id="0r5e1k"
public class Main {
    public static void main(String[] args) {

        int a = Integer.parseInt(args[0]);
        int b = Integer.parseInt(args[1]);

        System.out.println(a + b);
    }
}
```

Run:

```bash id="8n06bz"
java Main 10 20
```

Output:

```text id="bjlwm8"
30
```

# Why Do We Use `String[] args`?

We use command-line arguments when:

* input is needed BEFORE program execution
* automating scripts
* passing configuration/settings
* running programs from terminal/server
* competitive environments/tools
* batch processing

# Difference Between Scanner and `args`

| Feature       | `Scanner`          | `String[] args`  |
| ------------- | ------------------ | ---------------- |
| Input Time    | During runtime     | Before runtime   |
| Input Source  | Keyboard/System.in | Terminal command |
| Dynamic Input | Yes                | No               |
| Datatype      | Any                | String only      |
| Interaction   | User interactive   | Predefined input |

# Can Program Run Without `String[] args`?

# YES

You can write:

```java id="7nzyjv"
public static void main(String args[])
```

or:

```java id="s1tzm1"
public static void main(String... args)
```

Both are valid.

---

# But Can We Remove Parameters Completely?

## NO

This is INVALID:

```java id="9m3t9w"
public static void main()
```

JVM will NOT recognize it as the entry point.

Program will not run properly.

---

# Why JVM Needs It?

JVM is designed to call:

```java id="h6yqf6"
main(String[] args)
```

so it can pass command-line arguments into the program.

---

# Different Valid Main Signatures

## Valid

```java id="mjlwmz"
public static void main(String[] args)
```

```java id="9tb8kt"
public static void main(String args[])
```

```java id="v9k4hv"
public static void main(String... args)
```

---

# Invalid

```java id="34eb7u"
public static void main()
```

```java id="1qv9h6"
public void main(String[] args)
```

```java id="7j9s7v"
static void main(String[] args)
```

because JVM specifically looks for:

```java id="tpt6f2"
public static void main(String[] args)
```

# What Happens If No Arguments Are Passed?

Example:

```java id="i7olrz"
public class Main {
    public static void main(String[] args) {

        System.out.println(args.length);
    }
}
```

Run:

```bash id="lmm7uv"
java Main
```

Output:

```text id="muk18v"
0
```

So:

* `args` still exists
* but array size becomes `0`

---

# Common Error

```java id="rff2xk"
System.out.println(args[0]);
```

without passing arguments causes:

```text id="s4fpg5"
ArrayIndexOutOfBoundsException
```

because array is empty.
