
# **6. Input and Output in Java**

### 🔹 Definition

Input and Output (I/O) in Java refers to how a program **receives data (input)** and **displays results (output)**.

---

### ✅ Using `Scanner` Class

The `Scanner` class is the most commonly used way to take input from the user.

### 🔹 Features:

* Easy to use
* Supports different data types (`int`, `double`, `String`, etc.)
* Located in `java.util` package

### 🔹 Example:

```java
import java.util.Scanner;

public class InputExample {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int age = sc.nextInt();     // Takes integer input
        String name = sc.next();    // Takes single word input

        System.out.println(name + " is " + age);

        sc.close(); // Close the scanner
    }
}
```

### ⚠️ Important Notes:

* `next()` → reads a single word
* `nextLine()` → reads a full sentence
* Always close the scanner to prevent memory leaks

---

### ✅ Using `BufferedReader`

### 🔹 Definition:

`BufferedReader` is a faster way to read input compared to `Scanner`, but it requires manual parsing.

### 🔹 Example:

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.IOException;

public class BufferExample {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int age = Integer.parseInt(br.readLine());
        String name = br.readLine();

        System.out.println(name + " is " + age);
    }
}
```

### 🔹 Difference: Scanner vs BufferedReader

| Feature        | Scanner   | BufferedReader     |
| -------------- | --------- | ------------------ |
| Speed          | Slower    | Faster             |
| Ease of Use    | Easy      | Complex            |
| Parsing Needed | No        | Yes                |
| Best For       | Beginners | Competitive coding |

## **7. Type Casting**

### 🔹 Definition:

Type casting is the process of converting one data type into another.

---

### ✅ 1. Implicit Casting (Widening)

* Automatically done by Java
* Converts smaller type → larger type

```java
int myInt = 9;
double myDouble = myInt; // 9 → 9.0
```

### 🔹 Explanation:

No data loss occurs because the larger type can hold the value.

---

### ✅ 2. Explicit Casting (Narrowing)

* Done manually by programmer
* Converts larger type → smaller type

```java
double num = 9.78;
int result = (int) num; // 9 (decimal lost)
```

### ⚠️ Note:

* Data loss may occur
* Use carefully

## **8. Constants**

### 🔹 Definition:

Constants are variables whose values **cannot be changed once assigned**.

### 🔹 Keyword Used:

`final`

---

### 🔹 Example:

```java
final float PI = 3.14f;
// PI = 3.15f; // Error
```

### 🔹 Features:

* Must be initialized once
* Cannot be reassigned
* Naming convention: UPPERCASE

```java
final int MAX_USERS = 100;
```

## **9. Arrays**

### 🔹 Definition:

An array is a collection of **elements of the same data type stored in contiguous memory locations**.

---

### ✅ 1. Single-Dimensional Array

```java
int[] scores = {90, 80, 70};

System.out.println(scores.length); // 3
System.out.println(scores[0]);     // 90
```

---

### ✅ 2. Looping Through Arrays

#### 🔸 For Loop:

```java
for (int i = 0; i < scores.length; i++) {
    System.out.println(scores[i]);
}
```

#### 🔸 For-Each Loop:

```java
for (int i : scores) {
    System.out.println(i);
}
```

---

### ✅ 3. Two-Dimensional Array

```java
int[][] matrix = {
    {1, 2},
    {3, 4}
};

System.out.println(matrix[0][1]); // 2
```

## **10. Conditional Statements**

### 🔹 Definition:

Conditional statements allow decision-making in programs based on conditions.

---

### ✅ 1. If-Else Ladder

```java
int marks = 85;

if (marks > 90) {
    System.out.println("A");
} else if (marks > 80) {
    System.out.println("B");
} else {
    System.out.println("C");
}
```

### 🔹 Flow:

* Check conditions one by one
* First true condition executes
* Remaining are skipped

---

### ✅ 2. Switch Statement

```java
int day = 2;

switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    default:
        System.out.println("Invalid");
}
```

### 🔹 Keywords:

* `case` → possible values
* `break` → stops execution
* `default` → fallback option

---

## **11. Loops**

### 🔹 Definition:

Loops are used to execute a block of code repeatedly.

---

### ✅ 1. For Loop

```java
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
```

### 🔹 Best for:

When number of iterations is known

---

### ✅ 2. While Loop

```java
int i = 0;

while (i < 5) {
    System.out.println(i);
    i++;
}
```

### 🔹 Best for:

When condition is checked before execution

---

### ✅ 3. Do-While Loop

```java
int i = 0;

do {
    System.out.println(i);
    i++;
} while (i < 5);
```

### 🔹 Key Feature:

* Executes **at least once**, even if condition is false

---

## **12. Exception Handling**

### 🔹 Definition:

Exception handling is used to **handle runtime errors** and prevent program crashes.

---

### ✅ Structure:

```java
try {
    // Code that may cause error
} catch (Exception e) {
    // Handles error
} finally {
    // Always executes
}
```

---

### 🔹 Example:

```java
try {
    int[] myNumbers = {1, 2, 3};
    System.out.println(myNumbers[10]); // Error
} catch (Exception e) {
    System.out.println("Something went wrong.");
} finally {
    System.out.println("The 'try catch' is finished.");
}
```

---

### 🔹 Explanation:

* `try` → risky code
* `catch` → handles error
* `finally` → always runs
