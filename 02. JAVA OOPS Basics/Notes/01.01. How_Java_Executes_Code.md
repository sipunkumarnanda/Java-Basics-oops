
# 📘 How Java Executes Code — Complete In-Depth Notes

---

# 🌟 Introduction

One of the most important concepts in Java is understanding:

> 🔥 “How Java actually executes code internally?”

Most beginners think:

```text id="70r4u2"
Java reads code line-by-line and immediately executes it
```

❌ This is NOT fully correct.

Java execution happens in multiple stages:

* compilation
* parsing
* symbol resolution
* bytecode generation
* class loading
* JVM execution

Understanding this deeply helps in:
✅ debugging
✅ OOP understanding
✅ constructor chaining
✅ method calling
✅ interview preparation
✅ teaching Java properly

---

# 🎯 Big Picture of Java Execution

---

# 🔥 Java Execution Flow

```text id="d4zsmc"
Java Source Code (.java)
          ↓
Compilation by javac
          ↓
Bytecode (.class)
          ↓
Class Loader
          ↓
JVM Memory Allocation
          ↓
Bytecode Verifier
          ↓
Interpreter / JIT Compiler
          ↓
Machine Code
          ↓
Execution by CPU
```

---


file.

---

# 🔥 Command


# 🧠 What Happens?

Java compiler (`javac`):

* checks syntax
* checks semantic rules
* parses class structure
* creates symbol table
* resolves methods/constructors
* generates bytecode

# 📘 Output

```text id="hxyk8j"
Test.class
```

Bytecode is:
✅ intermediate code
✅ platform independent
✅ understood by JVM


because bytecode runs on JVM.

# 📘 Step 3 — Parsing Phase

This is where your important doubt comes from 🔥

---

# 🧠 Parsing Means

Compiler reads entire class structure first.

It stores:

* methods
* constructors
* variables
* signatures

---

# 🔥 Example

```java id="yitd1j"
class Test {

    void show() {

        display();
    }

    void display() {

        System.out.println("Hello");
    }
}
```

---

# ❓ Beginner Doubt

How can:

```java id="v3fjlwm"
display();
```

be called before method definition?

---

# ✅ Answer

Because compiler already parsed the full class before execution.

---

# 📘 Symbol Table Creation

Compiler internally creates something like:

```text id="2z2ih9"
Methods:
------------
show()
display()

Constructors:
------------
Test()
```

This internal structure is called:

# ✅ Symbol Table

---

# 🧠 Important Concept

Java execution is NOT:

```text id="n0q8xg"
Read line
Execute line
Read next line
```

Instead:

```text id="h3zj1x"
Read full class
Understand structure
Store metadata
Then execute
```

# 📘 Step 11 — Interpreter vs JIT Compiler

---

# 🔹 Interpreter

Reads bytecode line-by-line.

Slower.

---

# 🔹 JIT Compiler (Just-In-Time)

Converts frequently used bytecode into machine code.

Faster execution.

---

# 🌟 Why Java Is Fast Today?

Because modern JVM uses:
✅ JIT compiler
✅ optimization
✅ caching

# 📘 Difference Between Compilation & Execution

| Compilation          | Execution        |
| -------------------- | ---------------- |
| Checks syntax        | Runs program     |
| Creates bytecode     | Produces output  |
| Parses class         | Executes methods |
| Creates symbol table | Uses stack/heap  |

# 📘 Important Interview Questions

---

# ❓1. How Does Java Execute Code?

### ✅ Answer

Java source code is compiled into bytecode by `javac`, loaded by JVM, verified, converted into machine code by interpreter/JIT, and then executed by CPU.

---

# ❓2. Why Can Java Call Methods Written Below?

### ✅ Answer

Because compiler first parses the entire class and stores all method/constructor information before execution begins.

---

# ❓3. What Is Bytecode?

### ✅ Answer

Platform-independent intermediate code generated by Java compiler.

---

# ❓4. What Is JVM?

### ✅ Answer

Java Virtual Machine executes Java bytecode.

---

# ❓5. What Is Constructor Resolution?

### ✅ Answer

Process where Java identifies correct constructor based on passed arguments.

---

# ❓6. What Is Symbol Table?

### ✅ Answer

Internal compiler structure storing:

* methods
* constructors
* variables
* signatures

---

# ❓7. Why Are Local Variables Different?

### ✅ Answer

Because local variables are created during execution and do not exist before declaration.

---

# ❓8. Difference Between Heap and Stack?

| Heap               | Stack               |
| ------------------ | ------------------- |
| Stores objects     | Stores method calls |
| Shared memory      | Thread-specific     |
| Dynamic allocation | LIFO structure      |
