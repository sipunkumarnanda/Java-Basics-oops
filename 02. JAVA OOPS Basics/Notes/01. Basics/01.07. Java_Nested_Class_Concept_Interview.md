
# 📘 Java Nested Classes & Private Member Access — Complete Detailed Notes

# 🔥 Topic

# Why Can One Inner Class Access Another Inner Class’s `private` Members in Java?

---

# 📌 First Understand the Normal Rule of `private`

In Java:

```text id="f8g4h2"
private members are accessible only within the same class
```

That means:

* A private variable
* A private method
* A private constructor

cannot normally be accessed outside its class.

---

# 🔥 Example 1 — Normal `private` Rule

# ❌ Problem

```java id="d4k7m1"
class A {

    private int x = 10;
}

class B {

    void show() {

        A obj = new A();

        System.out.println(obj.x);
    }
}
```

---

# ❓ What Will Happen?

Will this compile?

---

# ❌ Output

```text id="w9v2n5"
Compilation Error
```

---

# ❌ Why Does It Fail?

Because:

```java id="q7r5c1"
x
```

is declared as:

```java id="a1f9x3"
private int x = 10;
```

inside class `A`.

And class `B` is a completely separate top-level class.

So this line becomes illegal:

```java id="k6u8t4"
System.out.println(obj.x);
```

because:

* `x` belongs to class `A`
* access is happening from class `B`

Java says:

```text id="m2j6p9"
private access is allowed only inside the same class
```

---

# 📘 Compiler Error Conceptually

```text id="h8s4v2"
x has private access in A
```

---

# 📊 Structure

```text id="z3n7b1"
A → separate top-level class
B → separate top-level class
```

No special relationship exists.

So access is denied.

---

# ✅ Correct Rule

| Situation                   | Private Accessible? |
| --------------------------- | ------------------- |
| Same class                  | ✅ Yes               |
| Different top-level classes | ❌ No                |

---

# 🔥 NOW — The Interesting Twist

Now look at this code carefully.

---

# 🔥 Problem

```java id="p5x2n8"
class Z {

    private int x = 20;

    class A {

        private int x = 10;
    }

    class B {

        void show() {

            A obj = new A();

            System.out.println(obj.x);
        }
    }
}
```

---

# ❓ What Will Be the Output?

Many programmers think:

```text id="v7c4k2"
❌ ERROR:
x is private inside A
So B should NOT access it
```

But surprisingly…

---

# ✅ Output

```text id="r1m8q4"
10
```

---

# 🤯 Why Does This Work?

This is one of Java’s most important nested class concepts.

Even though:

```java id="u2f6s9"
private int x = 10;
```

belongs to class `A`,

class `B` can still access it.

---

# 🔥 Main Reason

Because:

```text id="j4y9p3"
A and B are nested classes inside the same outer class Z
```

Java gives special access privileges to nested classes belonging to the same outer class.

---

# 📘 Important Java Rule

# ✅ Special Rule

```text id="g9n2v7"
Nested classes inside the same outer class
can access each other's private members.
```

---

# ❌ Normal Rule

```text id="e3r8k1"
private = accessible only within same class
```

---

# ✅ Special Exception

```text id="w6p1m4"
Nested classes inside same outer class
form a trusted relationship.
```

---

# 📊 Structure of the Program

```text id="c8u3x5"
Z (Outer Class)
│
├── private x = 20
│
├── class A
│     └── private x = 10
│
└── class B
      └── accesses A.x
```

Both:

* `A`
* `B`

belong to outer class:

```text id="f1q7s2"
Z
```

So Java allows access.

---

# 🧠 Step-by-Step Execution

---

# Step 1 → Create Object of A

```java id="d8m5n1"
A obj = new A();
```

An object of inner class `A` is created.

---

# Step 2 → Access `x`

```java id="k4v9p7"
System.out.println(obj.x);
```

Normally:

```text id="n6x2t8"
private members cannot be accessed outside class
```

BUT here:

* `A`
* `B`

belong to same outer class `Z`.

So Java allows it.

---

# ✅ Final Output

```text id="b2r5f9"
10
```

---

# 🔥 Which `x` Is Printed?

There are TWO variables named `x`.

---

# Outer Class Variable

```java id="s9m1q3"
private int x = 20;
```

belongs to class `Z`.

---

# Inner Class Variable

```java id="y4t8k6"
private int x = 10;
```

belongs to class `A`.

---

# Accessed Variable

```java id="u7n2p5"
obj.x
```

means:

```text id="x5c8r1"
x belonging to object obj of class A
```

So output becomes:

```text id="q2w9m7"
10
```

NOT:

```text id="v3k6s4"
20
```

---

# 🔥 Variable Shadowing Concept

This is also an example of:

# ✅ Variable Shadowing

because inner class variable hides outer class variable with same name.

---

# 📘 Accessing Outer Variable Explicitly

If you want outer class variable:

```java id="p1t7x9"
System.out.println(Z.this.x);
```

Output:

```text id="g4m8v2"
20
```

---

# 🔥 Internal Java Mechanism

Many interviewers ask this.

---

# ❓ How Does Java Actually Allow This?

Internally, Java compiler generates:

# ✅ Synthetic Accessor Methods

Example (conceptually):

```java id="n8s2w4"
static int access$100(A obj) {

    return obj.x;
}
```

These hidden methods allow nested classes to access private members safely.

---

# 📘 Important Compilation Detail

When compiled:

```text id="c3x7f1"
Outer.class
Outer$A.class
Outer$B.class
```

separate `.class` files are generated.

But Java still preserves special nested-class access rules.

---

# ✅ Output

# ✅ Why?

Outer class can access inner class private members.

Imagine:

# 🏠 House = Outer Class
