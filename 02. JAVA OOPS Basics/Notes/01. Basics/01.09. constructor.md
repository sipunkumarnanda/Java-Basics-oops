# 1️⃣ Parameterized Methods (Functions) 🧠

---

# 📌 What Is a Method?

A method is a block of code that performs a specific task.

Example:

```java id="m4x8p2"
void printHello() {

    System.out.println("Hello");
}
```

---

# 📌 What Is a Parameterized Method?

A method that accepts arguments/parameters is called:

# ✅ Parameterized Method

---

# 🔥 Definition

```text id="k9v2m5"
A method that accepts one or more arguments
is called a parameterized method.
```

---

# 📘 Example

```java id="q2n7x4"
class Sum {

    int add(int a, int b) {

        return a + b;
    }

    public static void main(String[] args) {

        Sum s = new Sum();

        int result = s.add(10, 20);

        System.out.println(result);
    }
}
```

---

# ✅ Output

```text id="p5m1v8"
30
```

---

# 🧠 Explanation

Method:

```java id="f8x3m7"
add(int a, int b)
```

accepts two parameters:

* `a`
* `b`

When called:

```java id="d4v9k1"
s.add(10, 20);
```

values are passed to method.

---

# 📌 Important Points

A method can have:

✅ no arguments
✅ one argument
✅ multiple arguments
✅ different data types

---

# 📘 Examples

---

# ✅ No Parameters

```java id="t7m2x6"
void hello() {

    System.out.println("Hello");
}
```

---

# ✅ One Parameter

```java id="u1p8m4"
void print(int x) {

    System.out.println(x);
}
```

---

# ✅ Multiple Parameters

```java id="v6n3x9"
void data(int id, String name) {

    System.out.println(id + " " + name);
}
```

# 2️⃣ Constructor in Java 🔥

---

# 📌 What Is a Constructor?

A constructor is a special block in Java used to:

```text id="j8x4m2"
initialize objects during object creation
```

---

# 🧠 Important Property

Constructor runs automatically when object is created.

---

# 📘 Example

```java id="z3m7p1"
class Test {

    public Test() {

        System.out.println("Default constructor is created");
    }

    public void printHello() {

        System.out.println("Hello");
    }
}
```

---

# 📘 Main Class

```java id="g6x2m9"
class Basics {

    public static void main(String[] args) {

        Test test = new Test();

        test.printHello();
    }
}
```

---

# ✅ Output

```text id="n1v8p5"
Default constructor is created
Hello
```

---

# 🧠 Step-by-Step Execution

---

# Step 1 → Object Creation

```java id="r4m9x2"
Test test = new Test();
```

---

# Step 2 → Constructor Executes Automatically

```java id="w7x3m1"
public Test()
```

runs automatically.

Output:

```text id="h5v2p8"
Default constructor is created
```

---

# Step 3 → Method Call

```java id="s8n4x6"
test.printHello();
```

prints:

```text id="f1m7v3"
Hello
```

---

# 📌 Important Rule

Constructor:

✅ same name as class
❌ no return type

---

# ❌ Wrong Constructor

```java id="x2m6p9"
void Test() {

}
```

This becomes a method, NOT constructor.

---

# 3️⃣ Default Constructor 🤖

---

# 📌 What Is Default Constructor?

If you do not create any constructor,

Java automatically provides one.

---

# Example

```java id="p7x2m5"
class Test {

}
```

Java internally creates:

```java id="c9m4v1"
Test() {

}
```

---

# 🧠 Why?

To initialize object properly.

---

# 4️⃣ Why Do We Need Constructors? 🔥

Constructors are needed when:

✅ object must start with initial values
✅ mandatory data is required
✅ object should be initialized automatically

---

# 📘 Real-Life Example — Bank Account 🏦

Different cities may require different minimum balances.

| City        | Minimum Balance |
| ----------- | --------------- |
| Kolkata     | ₹1000           |
| Bhubaneswar | ₹500            |
| Bangalore   | ₹2000           |

When opening account:

* minimum balance must be set
* account type must be initialized

Constructor helps automate this process.

---

# 📘 Bank Account Example

```java id="k1m8x4"
class BankAccount {

    double balance;

    String accountType;

    public BankAccount(double _balance) {

        this(_balance, "savings");
    }

    public BankAccount(double balance, String accType) {

        this.balance = balance;

        this.accountType = accType;
    }

    public void printAccType() {

        System.out.println("Account Type : " + this.accountType);
    }

    public void printBalance() {

        System.out.println("Balance : " + this.balance);
    }
}
```

---

# 🔥 Understanding Constructor Overloading

This class has:

# ✅ Two Constructors

---

# Constructor 1

```java id="m6v2x8"
public BankAccount(double _balance)
```

takes:

* only balance

and automatically sets:

```text id="q9x5m2"
accountType = savings
```

---

# Constructor 2

```java id="n4p8x1"
public BankAccount(double balance, String accType)
```

takes:

* balance
* account type

---

# 📌 This Is Called Constructor Overloading

---

# ✅ Definition

```text id="u7m3v9"
Multiple constructors with different parameter lists
is called constructor overloading.
```

---

# 🔥 Understanding `this()` Constructor Chaining

---

# Code

```java id="d2x7m5"
this(_balance, "savings");
```

---

# 📌 Meaning

Call another constructor of same class.

---

# 🧠 Why Used?

To avoid duplicate code.

---

# ❌ Without `this()`

```java id="b5m1x8"
balance = _balance;

accountType = "savings";
```

Duplicate logic.

---

# ✅ Better Approach

Reuse existing constructor.

---

# 📘 Main Program

```java id="f8v4m2"
public class Constructor {

    public static void main(String[] args) {

        BankAccount accPerson1 = new BankAccount(1000.00);

        accPerson1.printAccType();

        accPerson1.printBalance();

        System.out.println("---------------------------");

        BankAccount accPerson2 =
                new BankAccount(500.00, "current");

        accPerson2.printAccType();

        accPerson2.printBalance();
    }
}
```

---

# ✅ Output

```text id="z1m7p4"
Account Type : savings
Balance : 1000.0
---------------------------
Account Type : current
Balance : 500.0
```

---

# 🧠 Step-by-Step Flow

---

# 🔹 First Object

```java id="w4x9m6"
new BankAccount(1000.00)
```

calls:

```java id="j8m2v5"
BankAccount(double _balance)
```

which internally calls:

```java id="q5x1m8"
this(_balance, "savings");
```

Final values:

```text id="r3v7m1"
balance = 1000
accountType = savings
```

---

# 🔹 Second Object

```java id="y2m8x4"
new BankAccount(500.00, "current")
```

directly calls second constructor.

Final values:

```text id="u9x5m3"
balance = 500
accountType = current
```

# 📘 Understanding `this` Keyword 🔥

---

# 📌 `this`

refers to:

```text id="v1m6x9"
current object
```

---

# Example

```java id="k7x2m4"
this.balance = balance;
```

means:

```text id="c4m8v1"
current object's balance variable

local parameter balance
```

# 🔥 Constructor vs Method

| Feature                | Constructor       | Method            |
| ---------------------- | ----------------- | ----------------- |
| Purpose                | Initialize object | Perform operation |
| Name                   | Same as class     | Any name          |
| Return type            | ❌ No              | ✅ Yes             |
| Executes automatically | ✅ Yes             | ❌ No              |
| Called using object    | ❌ No              | ✅ Yes             |

---

# ❌ When NOT to Use Constructors

Avoid putting:

❌ heavy business logic
❌ database operations
❌ infinite loops
❌ file handling
❌ API calls

inside constructors.

---

# 🧠 Why?

Because constructor executes during object creation.

Heavy constructors make object creation slow.

---

# ✅ Constructors Should Mainly

```text id="n5v2x8"
initialize object state
```
