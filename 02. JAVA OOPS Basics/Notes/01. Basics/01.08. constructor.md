# Constructor in Java — Complete Guide

---

# 📌 What Is a Constructor in Java? 🤔

A constructor is a special block in Java that is used to:

```text id="f8x2m4"
initialize objects when they are created
```

It runs automatically whenever an object is created.

---

# 🧠 Simple Definition

# ✅ Constructor

```text id="v6n1p8"
A constructor is a special method-like block
used to initialize object variables.
```

---

# 📦 Real-Life Analogy 📱

Imagine buying a mobile phone.

When the phone is created/manufactured:

* RAM is set
* storage is set
* battery size is set

Similarly in Java:

```text id="x4r7m2"
constructor initializes object properties
```

when an object is created.

---

# 🔥 Why Do We Need Constructors?

Without constructors:

* objects may contain default values
* object initialization becomes repetitive
* code becomes messy

Constructors help:

✅ initialize variables automatically
✅ reduce repetitive code
✅ create fully initialized objects

---

# 📘 Syntax of Constructor

```java id="t9m3x6"
class ClassName {

    ClassName() {

        // initialization code
    }
}
```

---

# ⚠️ Important Rules of Constructor

| Rule                          | Explanation |
| ----------------------------- | ----------- |
| Constructor name = class name | ✅ Mandatory |
| No return type                | ✅ Important |
| Executes automatically        | ✅ Yes       |
| Called during object creation | ✅ Yes       |
| Can be overloaded             | ✅ Yes       |
| Cannot be static              | ❌ No        |
| Cannot be abstract            | ❌ No        |

---

# 🔥 First Example — Basic Constructor

```java id="p2v8m1"
class Student {

    int id;
    String name;

    Student() {

        id = 101;
        name = "Rahul";
    }

    void display() {

        System.out.println(id + " " + name);
    }

    public static void main(String[] args) {

        Student s = new Student();

        s.display();
    }
}
```

---

# ✅ Output

```text id="m7x4p2"
101 Rahul
```

---

# 🧠 Explanation

When:

```java id="k5n9v3"
Student s = new Student();
```

runs,

constructor automatically executes:

```java id="r1m8q7"
Student()
```

and initializes:

```text id="d4v2x9"
id = 101
name = Rahul
```

---

# 🔥 Constructor Executes Automatically ⚡

You never call constructor directly like normal methods.

Java automatically calls it during object creation.

---

# 📘 Constructor vs Method

| Feature                     | Constructor       | Method            |
| --------------------------- | ----------------- | ----------------- |
| Purpose                     | Initialize object | Perform operation |
| Name                        | Same as class     | Any valid name    |
| Return type                 | ❌ No return type  | ✅ Has return type |
| Called automatically        | ✅ Yes             | ❌ No              |
| Runs during object creation | ✅ Yes             | ❌ No              |

# 🔥 Types of Constructors in Java

Java mainly has:

1️⃣ Default Constructor
2️⃣ No-Argument Constructor
3️⃣ Parameterized Constructor
4️⃣ Copy Constructor (user-defined concept)

---

# 1️⃣ Default Constructor 🤖

If you do not create any constructor,

Java automatically provides one.

---

# Example

```java id="n8p2x5"
class Test {

    int x;

    void show() {

        System.out.println(x);
    }

    public static void main(String[] args) {

        Test t = new Test();

        t.show();
    }
}
```

---

# ✅ Output

```text id="u4m7v1"
0
```

---

# 🧠 Why 0?

Java automatically creates:

```java id="g9x1p3"
Test() {
}
```

called default constructor.

And instance variable gets default value:

```text id="w2r8m6"
int → 0
```

---

# 2️⃣ No-Argument Constructor 🛠️

Constructor created by programmer without parameters.

---

# Example

```java id="c7n5x2"
class Car {

    String color;

    Car() {

        color = "Black";
    }

    void show() {

        System.out.println(color);
    }

    public static void main(String[] args) {

        Car c = new Car();

        c.show();
    }
}
```

---

# ✅ Output

```text id="y6m1p4"
Black
```

---

# 3️⃣ Parameterized Constructor 🚀

Used when different objects need different values.

---

# Example

```java id="z4x8m2"
class Student {

    int id;
    String name;

    Student(int i, String n) {

        id = i;
        name = n;
    }

    void display() {

        System.out.println(id + " " + name);
    }

    public static void main(String[] args) {

        Student s1 = new Student(101, "Rahul");
        Student s2 = new Student(102, "Aman");

        s1.display();
        s2.display();
    }
}
```

---

# ✅ Output

```text id="r5n9v3"
101 Rahul
102 Aman
```

---

# 🧠 Explanation

Each object gets unique values through constructor parameters.

---

# 4️⃣ Copy Constructor 📋

Java does not provide built-in copy constructor like C++.

But we can create one manually.

---

# Example

```java id="m1x7p9"
class Student {

    int id;
    String name;

    Student(int id, String name) {

        this.id = id;
        this.name = name;
    }

    Student(Student s) {

        id = s.id;
        name = s.name;
    }

    void display() {

        System.out.println(id + " " + name);
    }

    public static void main(String[] args) {

        Student s1 = new Student(101, "Rahul");

        Student s2 = new Student(s1);

        s2.display();
    }
}
```

---

# ✅ Output

```text id="t6m3x8"
101 Rahul
```

# 🔥 Understanding `this` Keyword 🧠

Inside constructor:

```java id="f8n2v5"
this.id = id;
```

means:

```text id="p3x9m1"
current object's variable = local variable
```

---

# ⚠️ Variable Shadowing

```java id="h7m4x2"
Student(int id)
```

Here:

* parameter `id`
* instance variable `id`

have same name.

So:

```java id="b1v8p6"
this.id
```

refers to instance variable.

---

# 📘 Constructor Overloading 🔥

Multiple constructors with different parameter lists.

---

# Example

```java id="w5x2m9"
class Test {

    Test() {

        System.out.println("No-arg constructor");
    }

    Test(int x) {

        System.out.println("Parameterized constructor");
    }

    public static void main(String[] args) {

        Test t1 = new Test();

        Test t2 = new Test(10);
    }
}
```

---

# ✅ Output

```text id="z9n4p7"
No-arg constructor
Parameterized constructor
```

# 🔥 Constructor Chaining ⛓️

Calling one constructor from another constructor.

---

# Using `this()`

```java id="q2m8v4"
class Test {

    Test() {

        this(100);

        System.out.println("Default");
    }

    Test(int x) {

        System.out.println(x);
    }

    public static void main(String[] args) {

        Test t = new Test();
    }
}
```

---

# ✅ Output

```text id="x5p1m8"
100
Default
```

---

# 🧠 Rule

```text id="g8v3n1"
this() must be first statement inside constructor
```

# 🔥 Constructor and Inheritance 👨‍👦

Parent constructor executes before child constructor.

---

# Example

```java id="d6x2m7"
class Parent {

    Parent() {

        System.out.println("Parent Constructor");
    }
}

class Child extends Parent {

    Child() {

        System.out.println("Child Constructor");
    }

    public static void main(String[] args) {

        Child c = new Child();
    }
}
```

---

# ✅ Output

```text id="k1v9m4"
Parent Constructor
Child Constructor
```

---

# 🧠 Why?

Java automatically inserts:

```java id="m7p2x5"
super();
```

inside child constructor.

# 🔥 When Should We Use Constructors? ✅

Use constructors when:

✅ object initialization is required
✅ mandatory data must be set
✅ object should start in valid state
✅ values differ object-to-object
✅ dependency injection needed

---

# ✅ Example Use Cases

| Scenario                    | Use Constructor? |
| --------------------------- | ---------------- |
| Student object with id/name | ✅ Yes            |
| Bank account initialization | ✅ Yes            |
| Employee salary setup       | ✅ Yes            |
| Database connection setup   | ✅ Yes            |

---

# ❌ When NOT to Use Constructors

Avoid heavy logic inside constructors.

---

# ❌ Bad Practice

```java id="v9m4x1"
Test() {

    while(true) {

    }
}
```

---

# ❌ Avoid

* long loops
* file processing
* network calls
* heavy database operations
* complex business logic

inside constructors.

---

# 🧠 Why?

Because constructor runs during object creation.

Heavy constructors make object creation slow and risky.

---

# ✅ Better Approach

Use methods for heavy operations.

Constructors should mainly:

```text id="c4x8m2"
initialize object state
```

# 📘 Constructor Execution Order 🔥

Order during object creation:

```text id="r4x7m9"
1. Static block
2. Instance block
3. Constructor
```

---

# Example

```java id="j6m2v8"
class Test {

    static {

        System.out.println("Static Block");
    }

    {

        System.out.println("Instance Block");
    }

    Test() {

        System.out.println("Constructor");
    }

    public static void main(String[] args) {

        Test t = new Test();
    }
}
```

---

# ✅ Output

```text id="p9x4m2"
Static Block
Instance Block
Constructor
```
