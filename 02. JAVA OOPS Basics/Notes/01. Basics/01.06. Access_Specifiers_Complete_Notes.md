
# 📘 Java Access Specifiers — Complete In-Depth Notes (with Packages & Special Cases)

---

# 🌟 Introduction

Access specifiers (also called **access modifiers**) in Java control:

* visibility
* accessibility
* security
* encapsulation of data and methods

They define:

> “Who can access what?”

Java provides **4 access specifiers**:

| Access Specifier | Same Class | Same Package | Subclass (Different Package) | Other Packages |
| ---------------- | ---------- | ------------ | ---------------------------- | -------------- |
| `private`        | ✅          | ❌            | ❌                            | ❌              |
| default          | ✅          | ✅            | ❌                            | ❌              |
| `protected`      | ✅          | ✅            | ✅                            | ❌*             |
| `public`         | ✅          | ✅            | ✅                            | ✅              |

> ⚠️ `protected` members outside package are accessible only through inheritance.

---

# 🎯 Why Access Specifiers Are Important

They help achieve:

✅ Encapsulation
✅ Data Hiding
✅ Security
✅ Controlled Access
✅ Better OOP Design
✅ Modular Programming

---

# 🔐 1. `private` Access Specifier

---

# 📌 Definition

A `private` member is accessible:
✅ only inside the same class.

---

# 🧠 Key Features

* Most restrictive modifier
* Used for data hiding
* Cannot be inherited directly
* Cannot be accessed outside class

---

# ✅ Example

```java id="5ok9vt"
class Student {

    private int marks = 90;

    private void display() {
        System.out.println("Marks = " + marks);
    }

    public void show() {
        display();
    }
}

public class Main {

    public static void main(String[] args) {

        Student s = new Student();

        // System.out.println(s.marks); ❌ ERROR
        // s.display(); ❌ ERROR

        s.show();
    }
}
```

---

# ✅ Output

```text id="cgfx3g"
Marks = 90
```

---

# 🧠 Explanation

* `marks` and `display()` are private
* They cannot be accessed directly outside class
* Public method `show()` accesses them internally

---

# 🔥 Special Important Concept (Interview Favorite)

## ⭐ Private Members Between Inner Classes

Many students think:

> “private means inaccessible everywhere outside class”

But Java has a special rule for **nested/inner classes**.

---

# 🔥 Problem

```java id="5o3zw2"
class Z {

    private int x = 20;

    class A {

        private int x = 10;
    }

    class B {

        void show() {

            A obj = new A();

            System.out.println(obj.x);
        }
    }
}
```

---

# ❓ Question

How can class `B` access private variable `x` of class `A`?

---

# ✅ Output

```text id="8vsq35"
10
```

---

# 🧠 Deep Explanation

In Java:

> All inner classes inside the same outer class can access each other's private members.

Why?

Because compiler treats nested classes as part of the same top-level outer class environment.

---

# 📌 Important Rule

Inside outer class:

* inner classes can access private members of:

  * outer class
  * sibling inner classes

---

# 🔥 Visual Understanding

```text id="z1j1l4"
Outer Class Z
│
├── Inner Class A
│     private int x = 10
│
└── Inner Class B
      can access A.x ✅
```

---

# 🔥 Another Example

```java id="rgr3gx"
class Outer {

    class A {

        private int data = 100;
    }

    class B {

        void show() {

            A obj = new A();

            System.out.println(obj.data);
        }
    }

    public static void main(String[] args) {

        Outer o = new Outer();

        B b = o.new B();

        b.show();
    }
}
```

---

# ✅ Output

```text id="x9j2yk"
100
```

---

# 🧠 Why This Happens Internally

Java compiler generates synthetic accessor methods behind the scenes.

Example internally:

```text id="7ho0f3"
access$000()
```

These methods allow inner classes to access private members safely.

---

# ⚠️ Important Interview Point

## Q: Can two separate normal classes access each other's private members?

❌ NO

---

# Example

```java id="8db4ul"
class A {
    private int x = 10;
}

class B {

    void show() {

        A obj = new A();

        // System.out.println(obj.x); ❌ ERROR
    }
}
```

---

# 🧠 Reason

They are separate top-level classes.

Special access exists only for:
✅ nested/inner classes

# 📦 2. Default Access Specifier (Package-Private)

---

# 📌 Definition

If no modifier is specified:
Java uses default access.

Accessible only:
✅ inside same package

---

# ✅ Example

```java id="qnglmo"
class Test {

    int number = 10;

    void show() {
        System.out.println(number);
    }
}
```

---

# 📌 Accessible

| Location          | Access |
| ----------------- | ------ |
| Same package      | ✅      |
| Different package | ❌      |

---

# 🧠 Why Called Package-Private?

Because access is limited to package only.

---

# 🛡️ 3. `protected` Access Specifier

---

# 📌 Definition

Accessible:
✅ same package
✅ subclasses in different package

---

# ✅ Example

```java id="n8tt77"
class Animal {

    protected void sound() {

        System.out.println("Animal sound");
    }
}

class Dog extends Animal {

    void bark() {

        sound();
    }
}
```

---

# ✅ Output

```text id="3l8l6s"
Animal sound
```

---

# ⚠️ Important Rule

Outside package:

* protected accessible only via inheritance
* not through object reference

---

# ❌ Invalid Example

```java id="5ebc1z"
package pack2;

import pack1.A;

class B {

    void show() {

        A obj = new A();

        // obj.x ❌ protected not accessible
    }
}
```

---

# 🌍 4. `public` Access Specifier

---

# 📌 Definition

Accessible from everywhere.

---

# ✅ Example

```java id="b8fuvm"
public class Car {

    public void start() {

        System.out.println("Car started");
    }
}
```

```java id="e85jk5"
    public static void main(String[] args) {

    }
}
```

# ✅ Output

```text id="oaqdwx"

3
4
```

Inside same package:

* default ✅
* protected ✅
* public ✅
* private ❌

# 📊 Complete Access Table

| Modifier  | Same Class | Same Package | Subclass Different Package | Non-Subclass Different Package |
| --------- | ---------- | ------------ | -------------------------- | ------------------------------ |
| private   | ✅          | ❌            | ❌                          | ❌                              |
| default   | ✅          | ✅            | ❌                          | ❌                              |
| protected | ✅          | ✅            | ✅                          | ❌                              |
| public    | ✅          | ✅            | ✅                          | ✅                              |

---

# 🧠 Access Level Order

```text id="rbh3na"
private < default < protected < public
```

Most restrictive ➜ Least restrictive

# 🔥 Top-Level Class Rules

---

# ✅ Allowed

```java id="uxq7dw"
public class A {
}

class B {
}
```

---

# ❌ Not Allowed

```java id="6v6r6k"
private class A {
}

protected class B {
}
```

---

# 🧠 Why?

Top-level classes belong to package level.

So only:

* public
* default

are allowed.

# 🔥 Overriding Rules

You can:
✅ increase visibility

You cannot:
❌ decrease visibility

---

# ✅ Valid

```java id="5w1h3z"
class Parent {

    protected void show() {
    }
}

class Child extends Parent {

    public void show() {
    }
}
```

---

# ❌ Invalid

```java id="5l1w8n"
public → protected ❌
protected → default ❌
default → private ❌
```

# 🎯 Important Interview Questions with Answers

---

# ❓1. Difference Between Default and Protected?

| Default                        | Protected                               |
| ------------------------------ | --------------------------------------- |
| Same package only              | Same package + subclass outside package |
| No inheritance outside package | Supports inheritance                    |

---

# ❓2. Can Private Members Be Inherited?

❌ Directly no.

But subclass may indirectly access via public/protected methods.

---

# ❓3. Can Private Methods Be Overridden?

❌ No.

Because they are not inherited.

---

# ❓4. Why Use Private Variables?

For:
✅ encapsulation
✅ security
✅ controlled access

---

# ❓5. Why Can Inner Classes Access Private Members of Each Other?

Because Java compiler treats nested classes as part of same outer class scope and generates synthetic accessor methods internally.

---

# ❓6. Can Top-Level Classes Be Private?

❌ No.

Only nested classes can be private.

---

# ❓7. Which Access Modifier Is Most Secure?

✅ `private`

---

# ❓8. What Happens If No Access Modifier Is Written?

Java uses:
✅ default/package-private access

---

# ❓9. Which Modifier Gives Maximum Accessibility?

✅ `public`

---

# ❓10. Can Constructor Be Private?

✅ Yes.

Used in Singleton pattern.
