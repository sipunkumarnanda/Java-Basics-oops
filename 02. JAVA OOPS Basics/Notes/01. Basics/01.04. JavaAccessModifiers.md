
# Java Access Modifiers and Classes

Java provides four access levels:

1. public
2. protected
3. private
4. default (no modifier)

## Access Modifiers on Top-Level Classes

For a top-level class (a class defined directly in a `.java` file), only the following are allowed:

```java
public class Test { }
```

```java
class Test { }   // default (package-private)
```

The following are NOT allowed:

```java
private class Test { }   // Error
```

```java
protected class Test { } // Error
```

### Rule

A top-level class can be either:

* public
* default (no modifier)

It cannot be private or protected.

## Access Modifiers on Methods

Methods can use all access modifiers:

```java
public void m1() { }
protected void m2() { }
private void m3() { }
void m4() { }      // default
```

## Access Modifiers on Variables

Fields (instance variables) can also use all access modifiers:

```java
public int a;
protected int b;
private int c;
int d;             // default
```

## Inner (Nested) Classes

Unlike top-level classes, inner classes can use all access modifiers:

```java
public class Outer {

    private class Inner1 { }

    protected class Inner2 { }

    public class Inner3 { }

    class Inner4 { }
}
```

## Summary

| Modifier  | Top-Level Class | Method | Variable | Inner Class |
| --------- | --------------- | ------ | -------- | ----------- |
| public    | Yes             | Yes    | Yes      | Yes         |
| protected | No              | Yes    | Yes      | Yes         |
| private   | No              | Yes    | Yes      | Yes         |
| default   | Yes             | Yes    | Yes      | Yes         |

Key Point:
Top-level classes can only be public or default (package-private), while methods, variables, and inner classes can use all four access levels.

A simple memory trick:

**"Top-level Class → Public or Default only. Everything else → All four modifiers."**
