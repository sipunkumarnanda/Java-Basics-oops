
# 🚀 Java Classes, Objects & Access Specifiers — Complete Notes

---

# 📌 What is a Class in Java?

## ✅ Definition

A **Class** is a blueprint/template in Java that defines:

* Variables (data)
* Methods (functions)
* Behaviors
* Functionalities

Java is an **Object-Oriented Programming Language (OOP)**, so almost everything in Java is written inside a class.

---

# 🧠 Simple Understanding

Think of a class like:

# 🏠 House Blueprint

A blueprint contains:

* room design
* kitchen design
* doors/windows
* structure plan

But blueprint itself is NOT a real house.

Similarly:

```java id="oqov4u"
class Student {
}
```

is just a blueprint.

---

# 📦 Why Classes Are Needed?

Because Java organizes code using classes.

Everything you want to create:

✅ Variables
✅ Methods
✅ Logic
✅ Features
✅ Functionalities

must be inside a class.

---

# ✅ Example of Class

```java id="5n2v0z"
class Car {

    String color;

    void start() {
        System.out.println("Car Started");
    }
}
```

Here:

| Part      | Meaning         |
| --------- | --------------- |
| `color`   | Variable/Data   |
| `start()` | Method/Behavior |
| `Car`     | Blueprint/Class |

---

# 📌 What is an Object?

## ✅ Definition

An object is a real instance of a class.

Class = Blueprint
Object = Real thing created from blueprint

---

# 🏠 Real-Life Analogy

| Class         | Object       |
| ------------- | ------------ |
| Car blueprint | Actual car   |
| Student form  | Real student |
| Mobile design | Actual phone |

---

# ✅ Object Example

```java id="9tjlwm"
Car c1 = new Car();
```

Here:

| Part        | Meaning          |
| ----------- | ---------------- |
| `Car`       | Class            |
| `c1`        | Object reference |
| `new Car()` | Creates object   |

---

# 🔥 Complete Object Example

```java id="r4rkgg"
class AssignAge {

    int age;

    // Method to assign age
    public void assignAge(int num) {
        age = num;
    }
}

public class ClassesAndObjects {

    public static void main(String[] args) {

        // First object
        AssignAge x = new AssignAge();
        x.assignAge(10);

        System.out.println("X age is: " + x.age);

        // Second object
        AssignAge y = new AssignAge();
        y.assignAge(15);

        System.out.println("Y age is: " + y.age);
    }
}
```

---

# 🧠 Output

```text id="9ewvrv"
X age is: 10
Y age is: 15
```

---

# 🔍 Important Concept

```java id="7ad8r9"
AssignAge x = new AssignAge();
AssignAge y = new AssignAge();
```

creates:

✅ TWO different objects

Each object has separate memory.

---

# 📦 Memory Representation

```text id="mybl8z"
Object x → age = 10
Object y → age = 15
```

Both are independent.

# 🚀 Access Specifiers in Java

Access specifiers control:

# 🔐 "Who can access what?"

---

# 📌 Types of Access Specifiers

| Access Specifier        | Accessible Where?         |
| ----------------------- | ------------------------- |
| `public`                | Everywhere                |
| `private`               | Only inside same class    |
| `protected`             | Same package + subclasses |
| default/package-private | Same package only         |

---

# ✅ 1. Public Access Specifier

## Definition

`public` means accessible everywhere.

---

## Example

```java id="x3xy10"
public class Basics {
}
```

This class can be accessed from any package.

---

# 🌍 Real-Life Analogy

Public park 🌳

Everyone can access it.

---

# ✅ 2. Private Access Specifier

## Definition

`private` means:

Accessible ONLY inside current class.

---

# ✅ Example

```java id="m6lv1x"
public class ClassesAndObjects {

    private static void print(int num) {
        System.out.println(num);
    }

    public static void main(String[] args) {

        print(10);
    }
}
```

---

# 🔍 Why This Works?

Because:

```java id="jlwm9v"
print()
```

is being called inside same class.

So private access is allowed.

---

# ❌ Outside Access Not Allowed

```java id="f7efst"
class Demo {

    public static void main(String[] args) {

        ClassesAndObjects.print(10); // ERROR
    }
}
```

Because:

```text id="ny7ct1"
print() is private
```

and accessible only inside its own class.

---

# 🏠 Real-Life Analogy

Private bedroom 🚪

Only owner can access it.

---

# ✅ 3. Protected Access Specifier

## Definition

`protected` means:

Accessible:

✅ Inside same package
✅ In subclasses from other packages

---

# 🏠 Real-Life Analogy

Family property 👨‍👩‍👧

Family members can access it.

---

# ✅ 4. Default / Package-Private Access

## Definition

If no access specifier is given:

```java id="8rcd4o"
class Basics {
}
```

then Java automatically makes it:

# 📦 Package-Private

Meaning:

Accessible only inside same package.

---

# 🧠 Important Interview Point

```java id="k3s1pv"
class Basics
```

is NOT public.

It becomes:

```text id="cv1af7"
default/package-private
```

automatically.

---

# 🏠 Real-Life Analogy

Apartment building access 🏢

Only people inside building can access.

---

# 🚀 Static Keyword in Java

# 📌 What is Static?

`static` allows access without creating object.

---

# ✅ Example

```java id="l51c9e"
class Basics {

    static void mainMethod() {
        System.out.println("Hello");
    }
}
```

Call directly:

```java id="n00g4m"
Basics.mainMethod();
```

without object creation.

---

# 🧠 Why?

Because static methods belong to:

# 📦 Class level

not object level.

---

# 🔥 Non-Static vs Static

---

# ✅ Static Method

```java id="3lxbgm"
class Demo {

    static void show() {
        System.out.println("Static");
    }

    public static void main(String[] args) {

        show();
    }
}
```

No object needed.

---

# ✅ Non-Static Method

```java id="05zkml"
class Demo {

    void show() {
        System.out.println("Non Static");
    }

    public static void main(String[] args) {

        Demo obj = new Demo();

        obj.show();
    }
}
```

Object required.

---

# ❓ Why Non-Static Needs Object?

Because non-static methods belong to objects.

Java needs to know:

```text id="utv2i"
Which object's method should I call?
```

---

# ⚠️ Error Scenario

```java id="m3lln4"
class Demo {

    void show() {
        System.out.println("Hello");
    }

    public static void main(String[] args) {

        show(); // ERROR
    }
}
```

---

# ❌ Error

```text id="6wyqgd"
Cannot make a static reference to the non-static method
```

---

# ✅ Solution

Create object:

```java id="l8o8q8"
Demo obj = new Demo();

obj.show();
```

---

# 📌 Understanding main() Method

```java id="7skn88"
public static void main(String[] args)
```

---

# 🔍 Breakdown

| Keyword         | Meaning                         |
| --------------- | ------------------------------- |
| `public`        | JVM can access it from anywhere |
| `static`        | JVM can call without object     |
| `void`          | Returns nothing                 |
| `main`          | Entry point of Java program     |
| `String[] args` | Command line arguments          |

---

# 🚀 Why main() is Static?

JVM starts program execution using:

```java id="h7uqji"
main()
```

Before object creation.

So `main()` must be static.

Otherwise JVM would need object first.

---

void

# ❌ No return value

---

# ✅ Example

# 🚀 Important Interview Questions

# ❓Why is Java called Object-Oriented?

# ✅ Answer

Because Java organizes code using classes and objects, and supports OOP concepts like:

* Encapsulation
* Inheritance
* Polymorphism
* Abstraction

---

# ❓Difference Between Class and Object?

| Class               | Object             |
| ------------------- | ------------------ |
| Blueprint           | Real instance      |
| Logical entity      | Physical entity    |
| No memory allocated | Memory allocated   |
| Defines structure   | Stores actual data |

---

# ❓Can private methods be accessed outside class?

# ❌ No

Private methods are accessible only inside same class.

---

# ❓Can static methods access non-static methods directly?

# ❌ No

Because non-static methods need object reference.

---

# ❓Can non-static methods access static methods?

# ✅ Yes

Because static members already exist at class level.

# 🧾 One-Line Summary

> Java is an object-oriented language where classes act as blueprints, objects are real instances, access specifiers control visibility, and static members belong to class while non-static members belong to objects.
