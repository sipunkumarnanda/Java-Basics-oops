# 🎯 What is Polymorphism?

Polymorphism is one of the four pillars of OOP:

- 🔒 Encapsulation
- 🧬 Inheritance
- 🎭 Polymorphism
- 🎯 Abstraction

The word **Polymorphism** comes from:

```text
Poly     = Many
Morphism = Forms
```

👉 Meaning:

> One thing can exist in many forms.

---

## 📖 Definition

Polymorphism is the ability of a single interface, method, or reference variable to represent different behaviors depending on the object associated with it.

---

# 🎭 Simple Meaning of Polymorphism

Suppose you have:

```java
Animal a;
```

The same variable can refer to:

```java
Dog
Cat
Cow
Lion
Tiger
```

objects.

Example:

```java
Animal a;

a = new Dog();
a = new Cat();
a = new Cow();
```

Same variable.

Different forms.

This is Polymorphism.

---

# 🤔 Why Do We Need Polymorphism?

Without polymorphism:

```java
Dog d = new Dog();
Cat c = new Cat();
Cow cow = new Cow();
```

Need separate variables for each object.

---

With polymorphism:

```java
Animal a;
```

One variable can handle all animal objects.

Benefits:

✅ Less code

✅ Better flexibility

✅ Better maintainability

✅ Easier expansion

# 1️⃣ Compile-Time Polymorphism

Also called:

- Static Polymorphism
- Early Binding

Achieved through:

✅ Method Overloading

---

# 📖 What is Method Overloading?

Method Overloading occurs when:

- Same method name
- Different parameter list

---

## Example

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

---

## Main Method

```java
public class Main {

    public static void main(String[] args) {

        Calculator c = new Calculator();

        System.out.println(c.add(5, 10));

        System.out.println(c.add(5, 10, 15));
    }
}
```

---

## Output

```text
15
30
```

---

## How Java Decides?

Java checks:

```java
add(5,10)
```

Two arguments.

Therefore:

```java
int add(int a,int b)
```

is called.

---

When:

```java
add(5,10,15)
```

Three arguments.

Therefore:

```java
int add(int a,int b,int c)
```

is called.

---

## Why Called Compile-Time Polymorphism?

Because compiler decides:

```text
Which method should execute?
```

during compilation.

Not at runtime.

---

# 🎭 Method Overloading Rules

### Valid

```java
sum(int a, int b)

sum(int a, int b, int c)
```

---

### Valid

```java
sum(int a)

sum(double a)
```

---

### Invalid

```java
int sum(int a)

double sum(int a)
```

Only return type differs.

Compiler gets confused.

# 2️⃣ Runtime Polymorphism

Also called:

- Dynamic Polymorphism
- Late Binding

Achieved through:

✅ Method Overriding

✅ Inheritance

---

# 📖 What is Method Overriding?

When a child class provides its own version of a parent method.

---

# Example

## Parent Class

```java
class Animal {

    void sound() {
        System.out.println("Animal makes sound");
    }
}
```

---

## Child Class

```java
class Dog extends Animal {

    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}
```

---

Another Child:

```java
class Cat extends Animal {

    @Override
    void sound() {
        System.out.println("Cat meows");
    }
}
```

---

# 🔥 Main Program

```java
public class Main {

    public static void main(String[] args) {

        Animal a1 = new Dog();

        Animal a2 = new Cat();

        a1.sound();

        a2.sound();
    }
}
```

---

# Output

```text
Dog barks
Cat meows
```

---

# 🤯 Why Is This Amazing?

Look carefully:

```java
Animal a1 = new Dog();
Animal a2 = new Cat();
```

Reference Type:

```java
Animal
```

Actual Objects:

```java
Dog
Cat
```

---

When:

```java
a1.sound();
```

Java checks:

```text
What object is inside a1?
```

Answer:

```text
Dog
```

Execute:

```java
Dog.sound()
```

---

When:

```java
a2.sound();
```

Java checks:

```text
What object is inside a2?
```

Answer:

```text
Cat
```

Execute:

```java
Cat.sound()
```

---

# 🧠 Dynamic Method Dispatch

This mechanism is called:

## Dynamic Method Dispatch

Meaning:

Java decides at runtime:

```text
Which method should execute?
```

based on the actual object.

# 🎯 Polymorphism Using Interfaces

Polymorphism becomes even more powerful with interfaces.

---

## Interface

```java
interface Shape {

    void draw();
}
```

---

## Circle

```java
class Circle implements Shape {

    public void draw() {
        System.out.println("Drawing Circle");
    }
}
```

---

## Rectangle

```java
class Rectangle implements Shape {

    public void draw() {
        System.out.println("Drawing Rectangle");
    }
}
```

---

## Main

```java
public class Main {

    public static void main(String[] args) {

        Shape s;

        s = new Circle();
        s.draw();

        s = new Rectangle();
        s.draw();
    }
}
```

---

## Output

```text
Drawing Circle
Drawing Rectangle
```

# ✅ Advantages of Polymorphism

---

## 🔹 1. Code Reusability

Same code works for multiple objects.

---

## 🔹 2. Flexibility

New classes can be added easily.

Example:

```java
Dog
Cat
Cow
Lion
Tiger
```

No need to change existing code.

---

## 🔹 3. Maintainability

Less duplicate code.

Easier updates.

---

## 🔹 4. Extensibility

Code depends on interfaces instead of concrete classes.

---

## 🔹 6. Cleaner Code

One reference handles many object types.

# ❌ Disadvantages of Polymorphism

---

### 🔸 More Complex Design

Beginners may find it confusing.

---

### 🔸 Runtime Overhead

Method lookup takes a little extra time.

---

### 🔸 Debugging Difficulty

Finding actual method execution can be harder.

# 📊 Overloading vs Overriding

| Feature | Overloading | Overriding |
|----------|------------|------------|
| Polymorphism Type | Compile-Time | Runtime |
| Binding | Early | Late |
| Inheritance Required | ❌ No | ✅ Yes |
| Method Name | Same | Same |
| Parameters | Different | Same |
| Return Type | Can differ | Usually same/compatible |
| Decision Time | Compile Time | Runtime |
