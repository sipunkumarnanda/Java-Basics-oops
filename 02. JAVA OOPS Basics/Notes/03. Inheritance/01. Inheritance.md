

# 📘 1. Without Inheritance (Code Duplication)

### ❌ Example idea (no inheritance)

```java
class Car {
    String numberOfVehicle;

    public Car(String numberOfVehicle) {
        this.numberOfVehicle = numberOfVehicle;
    }

    public void honk() {
        System.out.println("Honk!!!!!!!");
    }

    public void printNumberOfVehicle() {
        System.out.println("Vehicle Number : " + this.numberOfVehicle);
    }
}

class Bus {
    String numberOfVehicle;

    public Bus(String numberOfVehicle) {
        this.numberOfVehicle = numberOfVehicle;
    }

    public void honk() {
        System.out.println("Honk!!!!!!!");
    }

    public void printNumberOfVehicle() {
        System.out.println("Vehicle Number : " + this.numberOfVehicle);
    }
}
```

### 🔴 Problems:

* Same variables repeated (`numberOfVehicle`)
* Same methods repeated (`honk()`, `printNumberOfVehicle()`)
* Hard to maintain (change needed in multiple places)
* Violates **DRY (Don’t Repeat Yourself)** principle

# 📘 2. With Inheritance (Your Code)

### ✅ Structure:

```java
class Vehicle {
    String numberOfVehicle;

    public Vehicle(String numberOfVehicle){
        this.numberOfVehicle = numberOfVehicle;
    }

    public void honk(){
        System.out.println("Honk!!!!!!!");
    }

    public void printNumberOfVehicle(){
        System.out.println("Vehicle Number : " + this.numberOfVehicle);
    }
}

class Car extends Vehicle {
    public Car(String numberOfVehicle){
        super(numberOfVehicle);
    }
}

class Bus extends Vehicle {
    public Bus(String numberOfVehicle){
        super(numberOfVehicle);
    }
}
```

### 🟢 Benefits:

* Common code written **once** in `Vehicle`
* `Car` and `Bus` **reuse** it
* Cleaner and shorter code
* Easy to maintain

### 1. ♻️ Code Reusability

Write common code once → reuse everywhere
👉 Example: `honk()` used by all vehicles

---

### 2. 🧹 Cleaner Code

Avoid repeating the same logic in multiple classes

---

### 3. 🔧 Easy Maintenance

Change code in one place (base class) → affects all child classes

---

### 4. 🧠 Real-World Modeling

Matches real-life relationships:

* Vehicle → Car, Bus
* Animal → Dog, Cat


# 📘 Inheritance in Java (Detailed Notes)

## 🔍 What is Inheritance?

Inheritance is a mechanism in
👉 **Object-Oriented Programming**
where one class **acquires the properties and behaviors** of another class.

* The class which is inherited → **Base Class / Parent Class / Superclass**
* The class which inherits → **Derived Class / Child Class / Subclass**

---

# 🏗️ Base Class vs Derived Class

### ✅ Base Class (Vehicle)

```java
class Vehicle {
    String numberOfVehicle;

    public Vehicle(String numberOfVehicle){
        this.numberOfVehicle = numberOfVehicle;
    }

    public void honk(){
        System.out.println("Honk!!!!!!!");
    }

    public void printNumberOfVehicle(){
        System.out.println("Vehicle Number : " + this.numberOfVehicle);
    }
}
```

### 📌 Explanation:

* Contains **common properties** → `numberOfVehicle`
* Contains **common methods** → `honk()`, `printNumberOfVehicle()`
* Acts as a **template** for other classes

---

### ✅ Derived Classes (Car, Bus)

```java
class Car extends Vehicle {
    public Car(String numberOfVehicle){
        super(numberOfVehicle);
    }
}

class Bus extends Vehicle {
    public Bus(String numberOfVehicle){
        super(numberOfVehicle);
    }
}
```

### 📌 Explanation:

* `extends Vehicle` → inherits all properties & methods
* No need to rewrite common code
* Can add **extra features** if needed

---

# 🔑 Important Concept: `extends`

* Used to create inheritance
* Syntax:

```java
class Child extends Parent
```

👉 Example:

```java
class Car extends Vehicle
```

---

# 🔑 Important Concept: `super`

### ✅ What is `super`?

* Refers to the **parent class object**
* Used to:

  1. Call parent constructor
  2. Access parent methods/variables

---

### 📌 In your code:

```java
super(numberOfVehicle);
```

### 🔍 What it does:

* Calls the **Vehicle constructor**
* Initializes `numberOfVehicle` in the parent class

---

# ⚙️ Object Creation Flow (Very Important ⭐)

### When you write:

```java
Car car = new Car("OD01A1234");
```

### 🔄 What happens internally:

1. Memory is allocated for `Car` object
2. Since `Car` **extends Vehicle**, it must also initialize `Vehicle` part
3. `super(numberOfVehicle)` is called
4. Parent constructor (`Vehicle`) executes first
5. Then `Car` constructor finishes

---

### 🧠 Key Point:

👉 **Parent class constructor is always called first**

Even if you don’t write `super()`, Java adds it automatically (default constructor case).

---

# 📦 Does it create two objects?

❌ No, **only ONE object is created**

👉 But it contains:

* Parent class part (Vehicle)
* Child class part (Car)

✔️ So:

> A `Car` object is also a `Vehicle`

---

# 🔁 Method Inheritance

```java
car.honk();
car.printNumberOfVehicle();
```

### 📌 Why this works?

* `Car` inherits methods from `Vehicle`
* So it can directly use them

### ✔️ Access modifiers matter

* `protected` / `public` → inherited
