# 1️⃣ Before Inheritance (Problem Statement)

## Code Without Inheritance

```java
class Car {
    private String numberOfVehicle;

    public Car(String numberOfVehicle) {
        this.numberOfVehicle = numberOfVehicle;
    }

    public void getCarNumber() {
        System.out.println("Car Number : " + this.numberOfVehicle);
    }

    public void honk() {
        System.out.println("Honk!!!!!!!");
    }
}

class Bus {
    private String numberOfVehicle;

    public Bus(String numberOfVehicle) {
        this.numberOfVehicle = numberOfVehicle;
    }

    public void getBusNumber() {
        System.out.println("Bus Number : " + this.numberOfVehicle);
    }

    public void honk() {
        System.out.println("Honk!!!!!!!");
    }
}

public class Inheritance2 {
    public static void main(String[] args) {

        Car car = new Car("OD01A1234");
        car.getCarNumber();
        car.honk();

        Bus bus = new Bus("OD02A1234");
        bus.getBusNumber();
        bus.honk();
    }
}
```

# 💡 Solution = Inheritance

Instead of writing common code multiple times, place it inside one parent class.

# 2️⃣ Inheritance Definition

## 📖 Definition

**Inheritance is an OOP concept where one class acquires the properties and behaviors of another class.**

---

### Parent Class

Also called:

* Super Class
* Base Class

```java
Vehicle
```

---

### Child Class

Also called:

* Derived Class
* Sub Class

```java
Car
Bus
```

---

# Real Life Example 🚗

Vehicle has:

* vehicle number
* honk()

Every Car has them.

Every Bus has them.

So:

```text
Vehicle
   ↑
 ┌─┴─┐
Car Bus
```

# 3️⃣ Code Using Inheritance

```java
class Vehicle {

    String numberOfVehicle;

    public Vehicle(String numberOfVehicle) {
        this.numberOfVehicle = numberOfVehicle;
    }

    public void honk() {
        System.out.println("Honk!!!!!!!");
    }

    public void printNumberOfVehicle() {
        System.out.println("Vehicle Number : " + this.numberOfVehicle);
    }
}
```

---

## Child Class

```java
class Car extends Vehicle {

    public Car(String numberOfVehicle) {
        super(numberOfVehicle);
    }
}
```

---

## Another Child Class

```java
class Bus extends Vehicle {

    public Bus(String numberOfVehicle) {
        super(numberOfVehicle);
    }
}
```

# 6️⃣ Constructor Execution

### Child Constructor

```java
public Car(String numberOfVehicle) {
    super(numberOfVehicle);
}
```

---

### super()

Calls Parent Constructor.

```java
super(numberOfVehicle);
```

actually means:

```java
Vehicle(numberOfVehicle);
```

---

Execution Flow:

```java
Car car = new Car("OD01A1234");
```

↓

```java
Car Constructor
```

↓

```java
Vehicle Constructor
```

↓

```java
numberOfVehicle initialized
```

↓

```java
Car object ready
```

---

# 7️⃣ Why super() is Needed

Suppose:

```java
Vehicle(String numberOfVehicle)
```

requires a parameter.

Without:

```java
super(numberOfVehicle);
```

compiler won't know how to initialize parent data.

---

Wrong ❌

```java
public Car(String numberOfVehicle) {

}
```

Error:

```text
Constructor Vehicle() is undefined
```

because parent has no default constructor.

# 8️⃣ Method Inheritance

Parent:

```java
public void honk() {
    System.out.println("Honk!!!!!!!");
}
```

---

Child:

```java
Car car = new Car("OD01A1234");

car.honk();
```

Output:

```text
Honk!!!!!!!
```

Even though Car does not define honk().

Because it inherits from Vehicle.

---

# 9️⃣ Method Overriding

Child can replace parent behavior.

Example:

```java
class Car extends Vehicle {

    public Car(String numberOfVehicle) {
        super(numberOfVehicle);
    }

    @Override
    public void honk() {
        System.out.println("Car Beep Beep");
    }
}
```

---

Now:

```java
car.honk();
```

Output:

```text
Car Beep Beep
```

Parent method overridden.

## 3. Hierarchical Inheritance

✅ Java Supports

## 4. Multiple Inheritance

```text
Parent1
    ↑
 Child
    ↑
Parent2
```

❌ Java Classes Don't Support

Reason:

Diamond Problem

---

### Java Supports Multiple Inheritance Through Interfaces

```java
interface A{}
interface B{}

class C implements A, B {}
```

✅ Allowed

# 1️⃣2️⃣ IS-A Relationship

Inheritance represents:

## IS-A Relationship

```text
Car IS-A Vehicle
Bus IS-A Vehicle
Truck IS-A Vehicle
```

---

Not:

```text
Engine IS-A Car
```

Wrong ❌

That is HAS-A relationship.

# 1️⃣3️⃣ HAS-A vs IS-A

## HAS-A

```java
class Engine {

}

class Car {
    Engine engine;
}
```

```text
Car HAS-A Engine
```

Composition

---

## IS-A

```java
class Car extends Vehicle
```

```text
Car IS-A Vehicle
```

Inheritance

# 🎯 Interview Questions & Answers

## Q1. What is Inheritance?

### Answer

Inheritance is an OOP concept in which one class acquires the properties and behaviors of another class for code reusability.

---

## Q2. Which keyword is used for inheritance in Java?

### Answer

```java
extends
```

Example:

```java
class Car extends Vehicle
```

---

## Q3. What is a Parent Class?

### Answer

A class whose properties are inherited by another class.

Example:

```java
Vehicle
```

---

## Q4. What is a Child Class?

### Answer

A class that inherits from another class.

Example:

```java
Car
```

---

## Q5. What is super()?

### Answer

`super()` calls the parent class constructor.

Example:

```java
super(numberOfVehicle);
```

---

## Q6. What is Method Overriding?

### Answer

Providing a new implementation of a parent method in child class.

Example:

```java
@Override
public void honk() {
    System.out.println("Beep Beep");
}
```

---

## Q7. Does Java Support Multiple Inheritance?

### Answer

❌ Through Classes → No

✅ Through Interfaces → Yes

---

## Q8. Difference Between Inheritance and Composition?

### Answer

| Inheritance      | Composition      |
| ---------------- | ---------------- |
| IS-A             | HAS-A            |
| extends          | Object Reference |
| Car IS-A Vehicle | Car HAS-A Engine |

---

## Q9. What are the benefits of inheritance?

### Answer

* Code Reusability
* Maintainability
* Extensibility
* Reduced Redundancy
* Better Organization

---

## Q10. What is the first thing executed in a child constructor?

### Answer

Parent constructor through:

```java
super()
```
