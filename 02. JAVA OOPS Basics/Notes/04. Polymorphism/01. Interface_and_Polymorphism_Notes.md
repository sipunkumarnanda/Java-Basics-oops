# 🎯 What is an Interface?

An **Interface** is a blueprint or contract that specifies **what a class must do**, but not **how it should do it**.

Think of an interface as a set of rules.

```java
interface Animal {
    void sound();
}
```

Here:

```java
void sound();
```

is only declared.

No implementation is provided.

The interface is saying:

> "If you want to become an Animal, you MUST provide a sound() method."

---

# 🏠 Real-Life Example: Remote Control

Suppose we have a TV remote.

The remote contains:

```text
Power On
Power Off
Volume Up
Volume Down
```

The remote only defines:

👉 What actions can be performed

It does NOT define:

👉 How each TV performs them

Different TVs perform these actions differently.

```text
Remote Control (Interface)

      ↓

Samsung TV
LG TV
Sony TV
```

All TVs follow the same rules.

# 🤔 Why Do We Need Interfaces?

Suppose we have:

```java
class Dog {
    void sound() {
        System.out.println("Dog barks");
    }
}

class Cat {
    void sound() {
        System.out.println("Cat meows");
    }
}
```

This works.

But imagine:

```text
Dog
Cat
Cow
Lion
Tiger
Elephant
Monkey
```

How can we treat all of them as Animals?

Solution:

👉 Create an Interface

```java
interface Animal {
    void sound();
}
```

Now every animal must follow the same contract.

---

# 🏗 Step 1: Create Interface

```java
interface Animal {
    void sound();
}
```

Meaning:

```text
Every Animal must have:
    ✓ sound()
```

---

# 🏗 Step 2: Implement Interface

## Dog

```java
class Dog implements Animal {

    public void sound() {
        System.out.println("Dog barks");
    }
}
```

---

## Cat

```java
class Cat implements Animal {

    public void sound() {
        System.out.println("Cat meows");
    }
}
```

---

## Cow

```java
class Cow implements Animal {

    public void sound() {
        System.out.println("Cow moos");
    }
}
```

Each class follows the contract but provides its own implementation.

# 🔥 Complete Example

```java
interface Animal {
    void sound();
}

class Dog implements Animal {

    public void sound() {
        System.out.println("Dog barks");
    }
}

class Cat implements Animal {

    public void sound() {
        System.out.println("Cat meows");
    }
}

class Cow implements Animal {

    public void sound() {
        System.out.println("Cow moos");
    }
}

public class Main {

    public static void main(String[] args) {

        Animal a;

        a = new Dog();
        a.sound();

        a = new Cat();
        a.sound();

        a = new Cow();
        a.sound();
    }
}
```

Output:

```text
Dog barks
Cat meows
Cow moos
```

# 🧠 How Does Java Decide Which Method to Execute?

When Java sees:

```java
a.sound();
```

it checks:

```text
What object is currently stored in a?
```

If:

```java
a = new Dog();
```

Call:

```java
Dog.sound()
```

If:

```java
a = new Cat();
```

Call:

```java
Cat.sound()
```

If:

```java
a = new Cow();
```

Call:

```java
Cow.sound()
```

Decision is made at:

# ⏰ Runtime

This is called:

## Runtime Polymorphism

or

## Dynamic Method Dispatch

# 🔥 Powerful Example Using Array

```java
interface Animal {
    void sound();
}

class Dog implements Animal {
    public void sound() {
        System.out.println("Dog barks");
    }
}

class Cat implements Animal {
    public void sound() {
        System.out.println("Cat meows");
    }
}

class Cow implements Animal {
    public void sound() {
        System.out.println("Cow moos");
    }
}

public class Main {

    public static void main(String[] args) {

        Animal[] animals = {
            new Dog(),
            new Cat(),
            new Cow()
        };

        for(Animal a : animals) {
            a.sound();
        }
    }
}
```

Output:

```text
Dog barks
Cat meows
Cow moos
```

# ✅ Advantages of Interface

### 1. Abstraction

Hide implementation details.

### 2. Loose Coupling

Classes depend on interfaces, not concrete classes.

### 3. Flexibility

Easy to replace implementations.

### 4. Multiple Inheritance

A class can implement multiple interfaces.

```java
class Student implements Sports, Music
```

Less code duplication.

### 🔹 Scalability

Large systems become easier to manage.

# 🎓 Interview Question

### Q1: Can we create an object of an interface?

❌ No

```java
Animal a = new Animal(); // Error
```

Interfaces cannot be instantiated.

---

### Q2: Why does this work?

```java
Animal a = new Dog();
```

Because Dog implements Animal.

Dog IS-A Animal.

---

### Q3: What type is `a`?

```java
Animal a = new Dog();
```

Reference Type:

```java
Animal
```

Object Type:

```java
Dog
```

# 📝 Exam Definitions

### Interface

> An interface is a blueprint or contract that defines a set of methods that implementing classes must provide.

### Polymorphism

> Polymorphism is the ability of a single reference variable to refer to objects of different classes and execute different behaviors using the same method call.

### Runtime Polymorphism

> Runtime polymorphism occurs when the method to be executed is determined at runtime based on the actual object referenced.
