---

# 🌟 What is Encapsulation?

Encapsulation is one of the core principles of OOP (Object-Oriented Programming).

It means:

> 🔐 Wrapping data (variables) and methods together into a single unit (class) and restricting direct access to data.

---

# 🎯 Main Goal of Encapsulation

✅ Data Hiding
✅ Security
✅ Controlled Access
✅ Better Maintainability
✅ Prevent Invalid Data Modification

---

# 🧠 Real-Life Example

## 🏦 Bank Account

You cannot directly modify your bank balance from outside.

Instead:

* bank controls access
* you use methods like:

  * deposit()
  * withdraw()
  * checkBalance()

This is encapsulation.

---

# 🔥 How Encapsulation Is Achieved in Java

Using:

✅ `private` variables
✅ public getter/setter methods

---

# 📌 Basic Structure

```java id="xw3gdx"
class Example {

    private int data;

    public void setData(int data) {
        this.data = data;
    }

    public int getData() {
        return this.data;
    }
}
```

---

# 🧠 Why Use `private`?

Because private variables:

* cannot be accessed directly outside class
* protect internal object state
* provide controlled access

---

# ❌ Without Encapsulation

```java id="v7tqax"
class Bank {

    public double balance;
}
```

Problem:

```java id="a5r9p7"
bank.balance = -999999;
```

Anyone can modify data incorrectly ❌

---

# ✅ With Encapsulation

```java id="4ycq1i"
private double balance;
```

Now direct access is blocked ✅

# 🔥 Complete Encapsulation Example

---

# ✅ Program

```java id="e1iijr"
// Class representing a bank account with an initial balance
class BankAccount {

    private double balance;
    private String accountType;

    // Constructor to initialize balance with default account type
    public BankAccount(double balance) {

        // Instead of writing:
        // this.balance = balance;
        // this.accountType = "Savings";

        // we call another constructor of the same class
        // using constructor chaining
        this(balance, "Savings");
    }

    // Main constructor
    public BankAccount(double balance, String accountType) {

        this.balance = balance;
        this.accountType = accountType;
    }

    // Setter method
    public void setBalance(double balance) {

        // Validation
        if(balance >= 0) {
            this.balance = balance;
        }
    }

    // Getter method
    public double getBalance() {

        // Add logic here to verify whether the user is authorized
        // before returning balance

        return this.balance;
    }

    // Helper method
    public void displayBalance() {
        System.out.println("Balance : " + this.balance);
    }
}

public class Encapsulation {

    public static void main(String[] args) {

        BankAccount person1 = new BankAccount(100);

        person1.displayBalance();

        /*
         * Attempting to change balance directly (not recommended)
         *
         * person1.balance = 0.00; ❌ ERROR
         *
         * Direct access breaks encapsulation.
         */

        person1.setBalance(500);

        System.out.println("Updated Balance : " + person1.getBalance());
    }
}
```

---

```text id="xho5kz"

Updated Balance : 500.0
```

# 🔥 Important Concept — Constructor Chaining

---

# 📘 What is Constructor Chaining?

Constructor chaining means:

> One constructor calls another constructor of the same class using `this()`.

---

# ✅ Example

```java id="n86z8l"
this(balance, "Savings");
```

means:

> “Call constructor having:
> `(double balance, String accountType)`”

---

# 🔥 Flow of Execution

When this runs:

```java id="7m2fok"
new BankAccount(100);
```

Java performs:

```text id="4z3tp6"
BankAccount(100)
        ↓
this(100, "Savings")
        ↓
BankAccount(100, "Savings")
        ↓
this.balance = 100
this.accountType = "Savings"
```

---

# 🧠 Why Constructor Chaining?

Benefits:

✅ Reduces duplicate code
✅ Cleaner initialization
✅ Easier maintenance
✅ Better readability

---

# 🔥 Main Constructor

```java id="0bgbom"
public BankAccount(double balance, String accountType)
```

is called:
✅ Main constructor

because actual initialization happens there.

---

# ❌ Common Mistake

---

# ❌ Wrong Code

```java id="qaq10x"
public BankAccount(double balance, String accountType) {

    this(balance, accountType);
}
```

---

# ❌ Problem

This constructor calls itself repeatedly:

```text id="9k9mj0"
Constructor
    ↓
Same Constructor
    ↓
Same Constructor
    ↓
Infinite recursion
```

---

# ❌ Result

```text id="ztr9rt"
StackOverflowError
```

---

# ✅ Correct Code

```java id="ye4esd"
public BankAccount(double balance, String accountType) {

    this.balance = balance;
    this.accountType = accountType;
}
```

---

# 🔥 Important Rule of `this()`

```java id="taz2h0"
this(...)
```

must always be:
✅ first statement inside constructor

---

# ❌ Invalid

```java id="m8m4o3"
public Test() {

    System.out.println("Hello");

    this(10); // ❌ ERROR
}
```

---

# ✅ Valid

```java id="hsx0db"
public Test() {

    this(10);

    System.out.println("Hello");
}
```

# 🔥 Constructor Overloading

---

# 📘 Definition

Multiple constructors with:

* same name
* different parameters

---

# ✅ Example

```java id="3g50l2"

BankAccount(double balance, String accountType)
```

---

# 🧠 Why Used?

To initialize objects in different ways.

---

# 🔥 Getter and Setter Methods

---

# 📘 Getter

Used to:
✅ return data

---

# ✅ Correct Getter

```java id="j4ktjr"
public double getBalance() {
    return this.balance;
}
```

---

# ❌ Wrong Getter

```java id="53u9hq"
public void getBalance() {
    System.out.println(balance);
}
```

---

# 🧠 Why Wrong?

Getter should:
✅ return value

not:
❌ print value

---

# 📘 Setter

Used to:
✅ modify data safely

---

# ✅ Example

```java id="v4rjgs"
public void setBalance(double balance) {

    if(balance >= 0) {
        this.balance = balance;
    }
}
```

---

# 🧠 Why Validation Is Important?

Prevents invalid data:

```java id="uy0rr4"
person1.setBalance(-50000);
```
