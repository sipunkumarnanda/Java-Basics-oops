# 🔐 Access Modifiers in Java

Access modifiers control:

# “Who can access what?”

Java has 4 access modifiers.

| Modifier        | Same Class | Same Package | Different Package (Subclass) | Different Package |
| --------------- | ---------- | ------------ | ---------------------------- | ----------------- |
| `private`       | ✅          | ❌            | ❌                            | ❌                 |
| package-private | ✅          | ✅            | ❌                            | ❌                 |
| `protected`     | ✅          | ✅            | ✅                            | ❌                 |
| `public`        | ✅          | ✅            | ✅                            | ✅                 |

---

# 🔒 1. `private`

Most restrictive access modifier.

---

# 📖 Definition

A private member is accessible:

# ✅ Only inside the same class

# 🧪 Example


# Understanding `private` vs Package-Private in Java (In Depth)

In Java, **access modifiers** control **visibility** — meaning:

> “From where can this class, variable, method, or constructor be accessed?”

Java provides four access levels:

| Access Modifier           | Same Class | Same Package | Different Package (Subclass) | Different Package (Non-Subclass) |
| ------------------------- | ---------- | ------------ | ---------------------------- | -------------------------------- |
| `private`                 | ✅          | ❌            | ❌                            | ❌                                |
| package-private (default) | ✅          | ✅            | ❌                            | ❌                                |
| `protected`               | ✅          | ✅            | ✅                            | ❌                                |
| `public`                  | ✅          | ✅            | ✅                            | ✅                                |

Your question is specifically about:

* `private`
* package-private (default access)

---

# What is `private`?

`private` is the **most restrictive access modifier** in Java.

When a member is declared `private`:

```java id="e7x6jd"
private int x;
```

it means:

> “This member can only be used inside the same class.”

No other class can directly access it.

---

# Example of `private`

```java id="3xg83h"
class Test {

    private int number = 100;

    private void display() {
        System.out.println("Display Method");
    }

    void accessInsideClass() {

        // Accessible because we are inside same class
        System.out.println(number);

        display();
    }
}
```

---

# Trying to Access from Another Class

```java id="mvx15v"
class X {

    public static void main(String[] args) {

        Test t = new Test();

        // ERROR
        // System.out.println(t.number);

        // ERROR
        // t.display();
    }
}
```

Why error?

Because:

* `number` is private
* `display()` is private

and private members belong only to the `Test` class itself.

---

# Real-Life Analogy of `private`

Think of a **mobile phone PIN**.

Only the owner of the phone can access it.

Other people cannot directly see or use it.

That is exactly how `private` works.

---

# Important Property of `private`

Even classes in the **same package** cannot access private members.

Example:

```java id="6c9s3x"
package mypack;

class A {
    private int x = 10;
}

class B {

    void show() {

        A obj = new A();

        // ERROR
        // System.out.println(obj.x);
    }
}
```

Even though:

* `A`
* `B`

are in same package,

`x` is still inaccessible because it is `private`.

---

# Why Use `private`?

`private` is mainly used for:

* Data hiding
* Security
* Encapsulation
* Protecting internal implementation

Example:

```java id="4fwz7w"
class BankAccount {

    private double balance;

    public void deposit(double amount) {
        balance += amount;
    }

    public double getBalance() {
        return balance;
    }
}
```

Here:

* User cannot directly change `balance`
* Only controlled methods can modify it

This prevents invalid operations.

# What is Package-Private (Default Access)?

When no access modifier is written:

```java id="ak04yb"

}
```

or

```java id="8x3nfd"
int x;
```

Java automatically gives:

# package-private access

also called:

* default access
* friendly access

---

# Meaning of Package-Private

It means:

> “Accessible to all classes inside the same package.”

But inaccessible outside the package.

---

# Example of Package-Private

```java id="shvl11"
package mypack;

class Test {

    int x = 50;

    void show() {
        System.out.println("Hello");
    }
}
```

Another class in same package:

```java id="rdzt3v"
package mypack;

class X {

    public static void main(String[] args) {

        Test t = new Test();

        System.out.println(t.x);

        t.show();
    }
}
```

Output:

```text id="knwmq9"
50
Hello
```

Accessible because both classes are inside same package.

---

# Different Package Example

```java id="9gqf1m"
package another;

import mypack.Test;

    public static void main(String[] args) {

        Test t = new Test();

        // ERROR
        // System.out.println(t.x);

        // ERROR
        // t.show();
    }
}
```

Why error?

Because package-private members are only visible inside the same package.

---

# Real-Life Analogy of Package-Private

Think of a college staff room.

Only teachers inside that department can access it.

People from other departments cannot.

Similarly:

* same package → allowed
* different package → not allowed

# Main Difference Between `private` and Package-Private

| Feature                        | `private`        | package-private  |
| ------------------------------ | ---------------- | ---------------- |
| Accessible inside same class   | ✅                | ✅                |
| Accessible inside same package | ❌                | ✅                |
| Accessible outside package     | ❌                | ❌                |
| Access level                   | Very restrictive | Less restrictive |
| Written as                     | `private`        | no modifier      |

# Why `private class Test` is Illegal?

Because a top-level class must be visible to the JVM/package system.

Java does not allow hiding a top-level class completely.

So this is invalid:

```java id="x2j7wq"
private class Test {

}
```

Compiler error:

```text id="6g6v1j"
modifier private not allowed here
```

---

# Where Can `private class` Be Used?

Only for inner/nested classes.

Example:

```java id="7ef7gr"
class Outer {

    private class Inner {

    }
}
```

Here:

* `Inner` can only be used inside `Outer`

This is valid.
