# 🌟 Introduction

In the previous part, we learned:

✅ Why an abstract class cannot be instantiated.

✅ Why abstract classes can have constructors.

✅ How Java initializes the parent portion of a child object.

Now let's answer another common interview question.

> **What members can an abstract class contain?**

Many beginners think:

> "An abstract class only contains abstract methods."

❌ This is incorrect.

An abstract class is almost like a normal class.

It can contain almost everything that a normal class can contain.

The only additional feature is that it **may also contain abstract methods.**

---

# 🏗️ Members Allowed in an Abstract Class

An abstract class can contain:

```text
📦 Instance Variables        ✅

📌 Static Variables          ✅

🏗 Constructors              ✅

⚙️ Concrete Methods          ✅

🏛 Abstract Methods          ✅

📌 Static Methods            ✅

🔒 Final Methods             ✅

👀 Nested Classes            ✅
```

The only thing it cannot do is

```java
new AbstractClass();
```

---

# 📦 Instance Variables

An abstract class can have instance variables just like a normal class.

Example

```java
abstract class BankAccount{

    String accountNumber;

    double balance;

}
```

Question:

Can child classes use these variables?

Yes.

```java
class SavingsAccount extends BankAccount{

    void display(){

        System.out.println(accountNumber);

        System.out.println(balance);

    }

}
```

Output

```text
SB1001

50000
```

---

# 🤔 Why Store Variables in the Parent?

Suppose every account has

* Account Number
* Balance
* Customer Name

Should every child declare them?

```java
class SavingsAccount{

    String accountNumber;

    double balance;

}
```

```java
class CurrentAccount{

    String accountNumber;

    double balance;

}
```

```java
class SalaryAccount{

    String accountNumber;

    double balance;

}
```

The same variables are repeated.

Instead,

move them into the parent.

```java
abstract class BankAccount{

    String accountNumber;

    double balance;

}
```

Now every child automatically inherits them.

Think of a vehicle.

```text

```

Instead of writing them in

```text

Truck
```

again and again,

we store them once in

```text

```

Every child automatically gets them.

# ⚙️ Concrete Methods

An abstract class can contain normal methods.

Example

```java
abstract class BankAccount{

    void showBankName(){

        System.out.println("State Bank");

    }

}
```

Child

```java
class SavingsAccount extends BankAccount{

}
```

Main

```java
SavingsAccount account =
        new SavingsAccount();

account.showBankName();
```

Output

```text
State Bank
```

---

# 🤔 Why Use Concrete Methods?

Suppose every account prints

```text
Welcome to SBI
```

Instead of writing

```java
void showBankName(){

}
```

inside every child,

we write it once.

Every child inherits it.

This reduces duplicate code.

# 🏛 Abstract Methods

Now comes the most important member.

```java
abstract void deposit(double amount);
```

Notice carefully.

There is

* No body
* No implementation

It only declares

```text
Every BankAccount

↓

Must Deposit Money
```

The implementation will be provided by child classes.

---

# 💻 Example

Parent

```java
abstract class BankAccount{

    abstract void deposit(double amount);

}
```

Child

```java
class SavingsAccount
        extends BankAccount{

    @Override
    void deposit(double amount){

        System.out.println("Savings Deposit");

    }

}
```

Current Account

```java
class CurrentAccount
        extends BankAccount{

    @Override
    void deposit(double amount){

        System.out.println("Current Deposit");

    }

}
```

Same method.

Different behavior.

This is also

✅ Runtime Polymorphism.

Access

# 📌 Static Methods

An abstract class can also contain static methods.

```java
abstract class BankAccount{

    static void displayBank(){

        System.out.println("SBI");

    }

}
```

Call

```java
BankAccount.displayBank();
```

Output

```text
SBI
```

Question

Can static methods be abstract?

No.

Example

```java
abstract static void show();
```

Compiler Error.

---

# 🤔 Why Can't Static Methods Be Abstract?

Static methods belong to the class.

Abstract methods belong to child objects.

Static methods are resolved at

```text
Compile Time
```

Abstract methods are resolved at

```text
Run Time
```

Since static methods cannot be overridden,

they cannot be abstract.

---

# 🔒 Final Methods

An abstract class may contain final methods.

Example

```java
abstract class BankAccount{

    final void bankRules(){

        System.out.println(
                "Minimum Balance Required");

    }

}
```

Child

```java
class SavingsAccount
        extends BankAccount{

}
```

Calling

```java
account.bankRules();
```

works perfectly.

---

# 🤔 Why Use Final Methods?

Suppose every account must follow RBI rules.

No child should modify them.

Then

```java
final
```

prevents overriding.

# 👀 Access Modifiers

Can abstract methods use access modifiers?

Yes.

```java
public abstract void deposit();
```

```java
protected abstract void deposit();
```

Package-private

```java
abstract void deposit();
```

All are valid.

---

# ❌ Can We Write

```java
private abstract void deposit();
```

No.

Compiler Error.

---

# 🤔 Why?

Private methods are

```text
Invisible

to

Child Classes
```

Abstract methods must be implemented

by

Child Classes.

If the child cannot even see the method,

how can it implement it?

Therefore

```java
private abstract
```

is illegal.

# 🤔 Why Can't We Write

```java
final abstract void deposit();
```

Think carefully.

Abstract means

```text
Must Override
```

Final means

```text
Cannot Override
```

These are opposite rules.

Therefore

```java
final abstract
```

is impossible.

Compiler Error.

# 🎯 Interview Questions

### Q1. Can an abstract class contain variables?

**Answer:**

Yes. It can contain both instance and static variables, just like a normal class.

---

### Q2. Can an abstract class contain concrete methods?

**Answer:**

Yes. Concrete methods allow common functionality to be shared among all subclasses.

---

### Q3. Can static methods be abstract?

**Answer:**

No. Static methods belong to the class and are resolved at compile time, whereas abstract methods require overriding and runtime resolution.

---

### Q4. Why is `private abstract` illegal?

**Answer:**

Because private methods are not inherited by child classes. Since abstract methods must be implemented by child classes, a private abstract method would be impossible to implement.

---

### Q5. Why is `final abstract` illegal?

**Answer:**

Because `abstract` requires a method to be overridden, while `final` prevents overriding. These two keywords contradict each other.

---

# ⭐ Key Takeaways

✅ An abstract class is almost a normal class.

✅ The only unique feature is that it may contain abstract methods.

✅ It can also contain variables, constructors, concrete methods, static methods, and final methods.

✅ Illegal combinations include:

* `private abstract`
* `static abstract`
* `final abstract`

Understanding **why** these combinations are illegal is more important than simply memorizing that they are illegal.
