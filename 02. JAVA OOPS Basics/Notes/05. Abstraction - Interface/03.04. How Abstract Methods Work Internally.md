> **How does Java execute an abstract method if the abstract class has no implementation?**

To answer this, we must understand how the **Java Virtual Machine (JVM)** works.


Many developers know **what** this code does.

# 🎤 Interview Questions

### Q1. How does JVM execute an abstract method?

**Answer:**

The abstract class provides only the method declaration. At runtime, the JVM looks at the actual object's class and invokes the overridden implementation in the concrete subclass using dynamic method dispatch.

---

### Q2. Does JVM call the abstract method?

**Answer:**

No.

The abstract method itself has no implementation.

The JVM always executes the overridden implementation provided by the concrete subclass.

---

### Q3. Where is method selection performed?

**Answer:**

Method selection for overridden instance methods is performed at **runtime** based on the actual object type. This is known as **runtime polymorphism** or **dynamic method dispatch**.

---

### Q4. Are two objects created?

**Answer:**

No.

Only one object—the concrete subclass object—is created. The inherited fields from the abstract class become part of that object.

---

# ⭐ Key Takeaways

✅ An abstract method contains only a declaration.

✅ The implementation is always provided by a concrete subclass.

✅ The JVM resolves the correct implementation at runtime.

✅ Abstract classes enable **abstraction**, while overridden methods and runtime dispatch enable **polymorphism**.

✅ A parent reference can refer to different child objects, allowing the same method call to produce different behavior.
